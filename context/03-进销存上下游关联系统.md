# 03-进销存上下游关联系统

## 一、核心关联系统与业务环节

强盛科技的进销存系统不是孤立运行的，它需要和以下外部系统或业务环节做数据交互：

| 关联方 | 关系类型 | 说明 |
|---|---|---|
| 华强北展示档口（门店） | 内部业务节点 | 零售收银的前端操作点，和民房仓库共享同一套库存底账 |
| 民房仓库（后仓） | 内部业务节点 | 库存实物存放点，负责拣货、发货、收货入库、盘点 |
| 30+ 地级市代理商（下游） | 外部客户 | 批发销售的对象，通过电话/QQ/微信下单，回款后需要登记核销 |
| 50+ 固定供应商（上游） | 外部供应商 | 采购下单的对象，送货后仓管点数入库，形成应付 |
| 财务记账 | 内部业务环节 | 收款付款登记、应收应付查询、基础对账——不是完整的财务总账系统 |

## 二、数据交互关系

| 关联方 | 输入给进销存 | 进销存输出给关联方 | 触发时机 |
|---|---|---|---|
| 门店（档口） | 零售成交记录、收款信息 | 实时库存数据、商品价格 | 每笔零售成交时 |
| 后仓（仓库） | 入库确认、出库确认、盘点结果、调拨确认 | 待出库清单、待入库清单、库存台账 | 每次库存动作执行时 |
| 代理商（下游客户） | 订单需求、回款 | 订单状态、应收账单 | 下单时、发货后、回款后 |
| 供应商（上游） | 发货通知、送货单 | 采购订单、应付账单 | 下单后、入库后 |
| 财务 | 收付款登记 | 应收应付数据、客户供应商往来 | 收付款登记时、月末对账时 |

## 三、系统边界

**属于进销存系统的能力（一期范围）：**
- 商品/客户/供应商/仓库主数据管理
- 销售订单 + 销售出库 + 库存占用
- 零售收银 + 库存扣减
- 采购订单 + 采购入库 + 采购退货
- 库存查询 + 调拨 + 盘点 + 报损
- 应收应付查询 + 收付款登记
- 基础销售/库存/往来查询统计

**不属于进销存系统的能力（外部系统或后续扩展）：**
- 电商平台接口（淘宝/京东等）→ 不在本期范围
- PDA 扫码仓储 → 不在本期范围
- 完整财务总账 → 不在本期范围，进销存只做到应收应付查询
- 自动支付流水认款 → 不在本期范围
- 代理商在线商城 → 不在本期范围
