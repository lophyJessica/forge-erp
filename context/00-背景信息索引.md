## AI 使用说明

当你被要求处理本项目相关任务时：
1. **先读本文件（00）**——了解有哪些 context 文件、各自解决什么问题、哪些还待填充
2. **按需读取后续文件**——根据任务类型选择性读取：
   - 写 PRD → 读 01, 02, 05, 07, 08 + templates/ 下全部模板
   - 画流程 → 读 02, 03, 06
   - 定义字段 → 读 05, 08 + templates/字段清单模板.md
   - 生成页面 → 读 01 §五（B端页面模式）+ templates/Demo PRD模板.md
   - 设计状态机 → 读 05 + templates/状态机设计模板.md + templates/单据状态与流转设计规范.md
3. **业务规则以 context/ 为准**——不要凭通用知识猜测。context/ 是最高权威

## 文件权威层级

context/ > templates/ > prd-docs/

- **context/**：业务事实、项目边界、设计约束——**最高权威**。如果 context 和 template 冲突，以 context 为准
- **templates/**：格式规范和输出模板——决定"怎么输出"
- **prd-docs/**：PRD 产出物——**生成物，不是权威源**。修改规则必须回溯到 context/ 或 templates/，不能直接在 PRD 里改规则
