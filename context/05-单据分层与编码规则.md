# 05-单据分层与编码规则

## 一、两层单据架构 + 财务结算环节

业务单据按「订单 → 执行」**两层**设计。财务结算作为独立环节挂接在执行层结果上，**不纳入单据分层**。

完整业务链路为「计划 → 执行 → 结算」，描述的是流程顺序，不等于「三层单据」。

### 1.1 两层业务单据

| 层级 | 名称 | 职责 | 采购模块举例 | 销售模块举例 |
|---|---|---|---|---|
| 第1层 | 业务订单层 | 记录意图和计划，不直接产生库存变动 | 采购订单 | 销售订单 |
| 第2层 | 业务执行层 | 记录实际发生的业务动作，直接影响库存 | 采购入库单、采购退货出库单 | 销售出库单、销售退货入库单 |

### 1.2 财务结算环节

| 环节 | 职责 | 触发方式 | 采购模块举例 | 销售模块举例 |
|---|---|---|---|---|
| 财务结算 | 记录资金往来与核销 | 执行层单据审核后自动生成应收/应付；收付款单独登记 | 应付记录、付款登记 | 应收记录、收款登记 |

**核心规则**：
- 执行层单据必须基于订单层单据创建（如采购入库单来源于采购订单）
- 订单层单据审核前可自由编辑，审核后关键字段锁定
- 库存变动只发生在执行层
- 应收/应付由执行层审核后自动生成，收付款登记用于核销往来，不属于业务单据分层

## 二、单据命名规则

### 单号前缀

| 单据类型 | 前缀 | 格式 | 示例 |
|---|---|---|---|
| 采购订单 | PO | `PO{YYYYMMDD}-{4位序号}` | PO20260523-0001 |
| 采购入库单 | PI | `PI{YYYYMMDD}-{4位序号}` | PI20260523-0001 |
| 采购退货单 | PR | `PR{YYYYMMDD}-{4位序号}` | PR20260523-0001 |
| 采购退货出库单 | PRO | `PRO{YYYYMMDD}-{4位序号}` | PRO20260523-0001 |
| 销售订单 | SO | `SO{YYYYMMDD}-{4位序号}` | SO20260523-0001 |
| 销售出库单 | SOO | `SOO{YYYYMMDD}-{4位序号}` | SOO20260523-0001 |
| 零售单 | RS | `RS{YYYYMMDD}-{4位序号}` | RS20260523-0001 |
| 调拨单 | TR | `TR{YYYYMMDD}-{4位序号}` | TR20260523-0001 |
| 盘点单 | CK | `CK{YYYYMMDD}-{4位序号}` | CK20260523-0001 |
| 报损单 | BL | `BL{YYYYMMDD}-{4位序号}` | BL20260523-0001 |
| 收款单 | RC | `RC{YYYYMMDD}-{4位序号}` | RC20260523-0001 |
| 付款单 | PY | `PY{YYYYMMDD}-{4位序号}` | PY20260523-0001 |
| 库存流水 | FL | `FL{YYYYMMDD}-{8位序号}` | FL20260523-00000001 |

> **库存流水说明**：由系统在业务单据确认/零售成交时自动生成，非用户手工创建；流水号不可修改、不回收。

### 编码规则

- 序号每日从 0001 开始，按创建时间递增
- 同一天、同一单据类型内序号不重复
- 审核后单号不可修改
- 作废单据的单号不回收

## 三、通用操作动词规范

| 规范用词 | 含义 | 禁止混用 |
|---|---|---|
| 新增 | 创建新单据 | 新建、创建、添加 |
| 编辑 | 修改已有记录 | 修改、更新、变更 |
| 查看 | 只读展示 | 详情、预览 |
| 停用 | 终止主数据可被新单引用 | 禁用、冻结、下架 |
| 启用 | 恢复主数据可被新单引用 | 激活、解冻、上架 |
| 取消 | 撤回单据（下游全部未执行时） | 作废 |
| 搜索 | 提交查询条件 | 查询（作为按钮文案时禁用） |

## 四、主数据通用规则

- 主数据（仓库、供应商、客户、商品）**不提供删除能力**
- 不再使用的对象通过"停用"操作处理
- 停用后不可在新建业务单据中选择，但历史单据仍保留完整数据
- 新增主数据时默认状态为"启用"，新增表单不展示状态字段
