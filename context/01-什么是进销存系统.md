# 01-什么是进销存系统

## 一、进销存是什么

进销存（JXC）是企业经营管理中最基础的三条业务线：**采购（进）、销售（销）、库存（存）**。

它解决的核心问题：企业买了多少货、卖了多少货、还剩多少货、钱收了多少、钱付了多少。进销存系统通常服务于**贸易型企业**——自己不生产，核心是买进来、卖出去、管好货。

## 二、进销存和 ERP 的关系

ERP（企业资源计划）范围更大，除了进销存外还包括财务总账、生产制造、人力资源、CRM 等。本项目的定位是**进销存系统**，聚焦采购、销售、库存、往来四条业务主线，不扩展到完整 ERP。

## 三、进销存的典型模块

| 模块 | 核心职责 | 典型单据/功能 |
|---|---|---|
| 基础资料 | 统一管理所有业务对象，全系统共用 | 商品档案、客户档案、供应商档案、仓库档案 |
| 采购管理 | 管"进"——从下单到入库到退货 | 采购订单、采购入库单、采购退货单、采购退货出库单 |
| 销售管理 | 管"销"——从订单到出库到退货 | 销售订单、销售出库单、销售退货单、销售退货入库单 |
| 零售管理 | 门店快速成交，轻量收银模式 | 零售收银、折让/抹零、小票打印 |
| 库存管理 | 管"存"——所有库存动作的统一底账 | 库存查询、调拨单、盘点单、报损单 |
| 往来管理 | 管"钱"——应收应付和收付款 | 应收查询、收款登记、应付查询、付款登记 |
| 查询统计 | 管"看"——基础经营数据查看 | 销售汇总、库存余额、客户往来查询 |
| 系统设置 | 系统运行的基础支撑 | 用户权限、单据编号、期初初始化、打印模板、操作日志 |

## 四、进销存的核心业务特征

- **单据驱动**：所有业务动作都以单据为载体（如采购订单→采购入库单→应付），不是口头沟通
- **库存是核心底账**：采购入库、销售出库、调拨、盘点、报损——所有动作最终都影响库存
- **数量 + 金额双口径**：不仅要管"有多少货"，还要管"值多少钱"。数量影响履约，金额影响往来
- **上下游联动**：采购入库影响库存增加 + 应付形成；销售出库影响库存减少 + 应收形成
- **两层单据 + 财务结算**：订单层（意图/计划）→ 执行层（实际发生）为两层业务单据；结算（往来/收付款）为独立环节，挂接在执行层结果上

## 五、B端管理后台的典型页面模式

AI 生成前端页面时，应遵循以下 B 端后台的常见模式（而非展示站或 C 端风格）：

### 页面骨架模式

| 页面类型 | 典型结构 | 说明 |
|---|---|---|
| 列表页 | 查询卡片 + 工具条 + 表格卡片 + 分页 | 最常用的页面类型，信息密度高、操作路径短 |
| 表单页 | 页头 + 表单卡片（头部信息 + 明细表格）+ 固定底部操作栏 | 新增/编辑共享同一套结构，通过路由参数区分 |
| 详情页 | 状态区 + 信息卡片 + 明细卡片 + 关联单据卡片 + 底部操作栏 | 只读展示，操作按钮按状态动态可用 |
| 作业页 | 作业说明 + 待处理明细 + 异常备注 + 底部确认 | 仓库/财务等执行类操作，强调效率和信息可见性 |
| 收银页 | 商品输入区 + 清单区 + 结算区 + 快捷操作 | 追求操作效率，布局紧凑 |

### 通用交互规范

- 所有状态变更通过**动作按钮**触发确认弹窗，**不在表单中直接修改状态字段**
- 主数据**不提供删除**，统一用"停用"替代。停用后不可新建引用，历史单据保留完整数据
- 单据审核后**关键字段锁定**，仅备注类字段仍可编辑
- 商品信息在单据中为**快照存储**，与商品档案变化解耦
- 必填字段标签前加红色星号 `*`
- 操作成功/失败用 Toast 反馈，阻断类校验用字段下方红色文字提示
- 固定底部操作栏的按钮顺序：次要操作左、主要操作右

### 控件选型指引

| 场景 | 推荐控件 | 说明 |
|---|---|---|
| 选择档案（供应商/客户/仓库） | `SelectWithSearch` | 支持按编码或名称搜索，展示格式 `{Code} {Name}` |
| 选择商品 | 商品选择弹窗 | 支持搜索+多选，选中后自动回填名称/规格/单位 |

| 批量搜索 | `BatchSearchInput` | 支持换行或逗号分隔多值粘贴 |
| 金额输入 | `NumberInput` | 前缀 `¥`，2 位小数，千分位展示 |
| 数量输入 | `NumberInput` | 正整数，不展示步进按钮 |
| 文本备注 | `Textarea` 或 `Input` | 根据预估字数选择，标注字符上限 |

## 六、进销存系统的关键设计原则

1. **两层单据架构 + 财务结算环节**：计划层（订单）→ 执行层（入库/出库）为两层业务单据；结算（应收应付、收付款）为独立环节。执行层单据必须基于订单层创建，库存变动只发生在执行层。
2. **库存三口径**：现存（实际有的）、占用（已锁定未出库）、可用（还能承诺的）。三个口径缺一不可。
3. **采购三数量**：采购数量（订单计划）、实收数量（实际到货）、入库数量（验收入库）。三个数量不一定相等，必须分开记录。
4. **价格与信用分离**：价格决定"卖多少"，信用决定"能不能卖"。一期先做价格体系，信用只做查询和提醒不做自动拦截。
5. **最小闭环优先**：一期不做复杂规则（补货建议、信用强控、审批流），先把"录得进、查得到、流转得通"做好。
