# 状态机生成

> **适用场景**：为某个单据设计或审核状态机。
> **前置准备**：对应的主 PRD + templates/状态机设计模板.md + templates/单据状态与流转设计规范.md

---

## 提示词

```
请为「[单据名称]」设计状态机，格式严格遵循 templates/状态机设计模板.md。

【输入】
- prd-docs/[路径]/[单据名]主PRD.md 中的业务场景和规则
- templates/状态机设计模板.md（格式规范）
- templates/单据状态与流转设计规范.md（设计原则）

【要求】
1. 状态定义表：每个状态注明枚举值、含义、进入条件、离开条件
2. Mermaid 状态流转图：覆盖所有状态的进入和离开路径
3. 动作矩阵：每个状态 × 每个动作，标注允许/不允许/需确认
4. 设计自检：按模板中的检查项逐一核验

【约束】
- 状态变更必须通过动作按钮触发，不能在表单中直接修改状态字段
- 先少后多：一期用够用的状态，不为显得专业而拆过细
- 状态和动作必须成对设计，不能有"到了某个状态不知道该干什么"
```
