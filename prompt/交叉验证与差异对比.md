# 交叉验证与差异对比

> **适用场景**：AI 产出后，和已有的人工产物做差异对比。
> **前置准备**：AI 产出文件 + 人工参照物。

---

## 提示词

```
请对比以下两份文档：

A — [人工产出的文件路径]
B — [AI 刚生成的文件路径]

找出 A 和 B 在以下维度的差异：

1. **结构差异**——B 比 A 多了哪些章节？少了哪些？顺序是否一致？
2. **状态机差异**——状态定义粒度、状态转换条件是否对称？
3. **业务规则差异**——A 中有但 B 中遗漏的规则，或 B 补充了 A 没有的规则
4. **术语一致性**——同一概念是否在两份文档中用了不同名称？
5. **格式规范性**——表格结构、Mermaid 语法、枚举值表达方式是否一致？

只列差异，不修改任何文件。每条差异标注「建议对齐」或「可接受差异」。
```

---

## 提示词 B：多份 AI 产出一致性检查

```
请对比以下 AI 生成的文档，检查它们之间的一致性：

[文件1路径]
[文件2路径]
[文件3路径]

检查维度：
- 同一字段在三份文档中的定义是否一致
- 同一状态在 PRD 和 Demo PRD 中的枚举值是否一致
- 同一业务规则在不同文档中的表述是否一致
- 是否存在"主PRD 说有、Demo PRD 里没体现"的遗漏

只列不一致的地方，每处标注建议的权威来源（以哪个文件为准）。
```
