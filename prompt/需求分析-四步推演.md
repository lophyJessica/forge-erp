# 需求分析：四步推演

> **适用场景**：拿到一个模糊需求，在写 PRD 之前先和 AI 做结构化推演。
> **前置准备**：context/ 下至少 3 个文件 + AGENTS.md 已就位。

---

## 提示词

```
请读取 context/ 下所有文件和 AGENTS.md。

我要为 [模块名称] 做需求分析。按以下四步帮我推演，每步输出后等我确认再继续下一步：

第一步：梳理功能全景
- 这个模块涉及哪些单据/功能点？
- 和上下游模块的数据依赖关系是什么？
- 一期范围判断：哪些做、哪些明确不做？

第二步：推演业务流程
- 画出核心业务链路的主流程（正常路径）
- 补充关键异常分支（至少 3 个）
- 用 Mermaid flowchart 输出

第三步：定义状态和规则
- 核心单据的状态机（状态定义 + 流转图 + 动作矩阵）
- 关键业务规则（校验规则、计算规则、状态约束）
- In Scope / Out of Scope 清单

第四步：汇总约束文档
- 把所有业务约束汇总成一份"XX模块边界与约束"文档
- 后续写 PRD 时直接喂入这份文档作为约束输入

每步输出后，告诉我：哪些地方你可能漏了或编了、需要我补充什么信息。
```
