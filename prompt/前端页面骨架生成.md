# 前端页面骨架生成

> **适用场景**：基于 PRD 生成 B 端管理后台前端页面。
> **前置准备**：目标模块的完整 PRD（主 PRD + 字段清单 + Demo PRD）。

---

## 提示词

```
请基于以下 PRD 文档，为「[模块名称]」生成前端页面骨架。

【输入】
- prd-docs/[路径]/[单据名]主PRD.md
- prd-docs/[路径]/[单据名]字段清单.md
- prd-docs/[路径]/[单据名]_Demo_列表页.md
- prd-docs/[路径]/[单据名]_Demo_新增编辑页.md
- prd-docs/[路径]/[单据名]_Demo_详情页.md

【技术栈】
React + TypeScript + Tailwind CSS + shadcn/ui

【输出要求】
生成以下文件，保存到 front-prototype/src/pages/ 目录：

1. **[PageName]List.tsx** — 列表页
   - 筛选区：所有可筛选字段 + 搜索/重置按钮
   - 表格：所有列表展示列 + 操作列（按钮根据状态动态可用）
   - 底部分页

2. **[PageName]New.tsx** — 新建/编辑页
   - 表单区：头部字段 + 明细表格（可增删行）
   - 自动计算字段（实时更新）
   - 底部操作按钮（保存草稿/提交）

3. **[PageName]Detail.tsx** — 详情页
   - 状态区：Badge + 时间线
   - 信息区：所有详情展示字段（只读）
   - 关联单据区（如有）
   - 底部操作按钮（根据状态动态展示）

【关键约束】
- 页面风格为 B 端管理后台——信息密度高、操作直接，不是展示站风格
- 状态 Badge 颜色：从字段清单/Demo PRD 读取各状态的枚举值和对应颜色，不使用默认状态名。常见配色参考：草稿=gray、待处理=blue、执行中=yellow、已完成=green、已关闭=red（以字段清单中的实际状态名为准）
- 金额字段右对齐，保留 2 位小数
- 日期格式 YYYY-MM-DD
- 状态变更通过按钮触发弹窗确认，不在表单中直接修改
- 字段名、枚举值与字段清单严格一致
```
