# PRD 生成：业务单据

> **适用场景**：已有 context/ 和 templates/，需要为某个业务单据生成完整 PRD。本提示词为通用版，适用于正式项目。
> **第 4 节演示用**：第 4 节「全流程预览课」使用精简版提示词（见 `演示提示词/05-生成采购入库单PRD.md`），输出定位为"初稿"而非"完整 PRD"。
> **前置准备**：context/ 全部文件 + templates/业务单据PRD模板.md + templates/字段清单模板.md + templates/状态机设计模板.md + templates/Demo PRD模板.md

---

## 提示词

```
请基于以下输入，为「[单据名称]」生成完整的 PRD 文档套件：

【喂入文件】
- context/ 下全部文件（业务背景和约束）
- templates/业务单据PRD模板.md（PRD 章节结构）
- templates/字段清单模板.md（字段定义格式）
- templates/状态机设计模板.md（状态机格式）
- templates/Demo PRD模板.md（页面规格格式）
- templates/单据状态与流转设计规范.md（状态设计原则）
- prd-docs/[参考模块]/[参考PRD].md（格式参照）

【输出要求】
生成以下文件，保存到 prd-docs/[模块名]/[单据名]/ 目录：

1. **[单据名]主PRD.md**
严格按照业务单据PRD模板的章节结构，包含：
- 业务背景
- 功能范围（In/Out Scope）
- 单据定位（层级、来源、下游、实体关系、系统链路 Mermaid 图）
- 业务场景（至少 4 个：主流程 + 关键支线 + 异常场景）
- 状态机（状态定义表 + Mermaid 状态流转图 + 动作矩阵表）
- 业务规则（校验规则、计算规则、状态约束）
- 页面与交互说明（列表页/新增编辑页/详情页各自的目标和关键要素）
- 验收标准

2. **[单据名]字段清单.md**
严格按照字段清单模板的格式，包含：
- 头部字段（单据级）
- 商品/明细字段（行级）
- 统计汇总字段
- 系统字段
- 展示与引用规则

3. **[单据名]_Demo_列表页.md** — 按 Demo PRD 模板
4. **[单据名]_Demo_新增编辑页.md** — 按 Demo PRD 模板
5. **[单据名]_Demo_详情页.md** — 按 Demo PRD 模板

【关键约束】
- 术语体系以 context/05-单据分层与编码规则.md 为准
- 状态变更通过动作按钮触发，不在表单中直接修改状态字段
- 主数据不提供删除，统一用"停用"替代
- 字段清单是字段定义的唯一权威源，Demo PRD 只引用字段名称
- 文档风格与参照 PRD 保持一致
```
