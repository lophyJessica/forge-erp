# 05-并行生成库存模块 PRD 套件

> 演示步骤：第 5 步（PRD 产出链路——**效率展示**）
> 策略：快讲——同一套提示词改模块名，2 个 Agent 并行跑，展示复用效率

## 喂入文件

两个 Agent 共享同一套输入：

@context/（全部：00-08）
@templates/业务单据PRD模板.md
@templates/字段清单模板.md
@templates/状态机设计模板.md
@templates/Demo PRD模板.md
@templates/单据状态与流转设计规范.md
@templates/业务流程推演模板.md
@templates/用例数据推演模板.md
@prd-docs/采购管理/采购入库单/（刚才生成的完整 PRD 套件）  ← 风格参照

## 提示词（直接复制——两个 Agent 各用一份，只改模块名和关键约束）

### Agent 1：库存流水查询

```
请基于以下输入，为「库存流水查询」生成完整的 PRD 套件初稿，保存到 prd-docs/库存管理/库存流水查询/ 目录下。

这是一套 PRD 产出物，不是一份文档。每份文件有且只有它该管的信息，互不越界。

参照物：prd-docs/采购管理/采购入库单/ 下的完整 PRD 套件——文档结构、表格格式、描述风格全部对齐。

输出文件：
1. 库存流水查询主PRD.md —— 需求背景、功能范围、单据定位、业务场景、业务规则、验收标准
2. 库存流水查询字段清单.md —— 查询条件字段 + 列表展示字段 + 系统字段
3. 库存流水查询_业务规则规格.md —— 查询权限规则、数据范围规则、筛选逻辑
4. 库存流水查询_业务流程推演.md —— 按商品查/按仓库查/按单据追溯/按时间范围查，含无数据场景
5. 库存流水查询_用例数据推演.md —— 带具体数据验证查询结果是否正确
6. 库存流水查询_Demo_列表页.md —— 筛选区（商品/仓库/单据类型/日期范围/变动方向）+ 表格列 + 导出按钮

关键约束：
- 库存流水是查询页面，不是操作页面——没有状态机，没有编辑表单
- 库存变动必须可追溯——每条流水记录关联来源单据号
- 库存变动方向：入库（+）、出库（-）
- 支持按商品、仓库、单据类型、时间范围组合筛选
- 术语体系以 context/08 为准
```

### Agent 2：即时库存查询

```
请基于以下输入，为「即时库存查询」生成完整的 PRD 套件初稿，保存到 prd-docs/库存管理/即时库存查询/ 目录下。

这是一套 PRD 产出物，不是一份文档。每份文件有且只有它该管的信息，互不越界。

参照物：prd-docs/采购管理/采购入库单/ 下的完整 PRD 套件——文档结构、表格格式、描述风格全部对齐。

输出文件：
1. 即时库存查询主PRD.md —— 需求背景、功能范围、单据定位、业务场景、业务规则、验收标准
2. 即时库存查询字段清单.md —— 查询条件字段 + 列表展示字段 + 系统字段
3. 即时库存查询_业务规则规格.md —— 查询权限规则、即时库存计算规则
4. 即时库存查询_业务流程推演.md —— 按商品查/按仓库查/按批次查/库存预警，含无库存场景
5. 即时库存查询_用例数据推演.md —— 即时库存 = 期初 + 所有入库 - 所有出库，带数据验证
6. 即时库存查询_Demo_列表页.md —— 筛选区（商品/仓库/批次/库存范围）+ 表格列 + 导出按钮

关键约束：
- 即时库存是查询页面，不是操作页面——没有状态机，没有编辑表单
- 即时库存 = 期初库存 + 所有入库 - 所有出库（实时计算，不是定时快照）
- 支持按商品、仓库、批次组合筛选
- 库存数量 ≤ 0 的可以设置"不显示零库存"开关
- 术语体系以 context/08 为准
```

## 讲师备注

**预期产出**：
- 两个模块各 6 份文件，同时产出
- 库存流水查询——列表页型 PRD，无状态机
- 即时库存查询——查询页型 PRD，无状态机，含实时计算逻辑

**点评要点**：
- 强调效率：同一套提示词结构，改了模块名和关键约束，两个 Agent 并行跑——几分钟出两个模块的完整 PRD 套件
- AI 正确判断了两个模块都是查询型页面，不需要状态机——这是它读了 context/ 后的自主判断
- 传统方式下两个模块的 PRD 套件可能要写一整天——现在几分钟出初稿
- 地基+模板+参照物+并行 Agent = 生产力乘法
