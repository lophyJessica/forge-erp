# 07-走查自检与归档

> 演示步骤：第 6 步 + 第 8 步（分两次使用：PRD 粗验 + 前端粗验+归档）
> 策略：PRD 产出后先用前半部分做验证关卡，前端产出后用后半部分做终验归档

## 前半部分：PRD 粗验（第 6 步使用）

> 四个模块的 PRD 套件都生成后，先做一次 PRD 自查——过不了关不往下走前端。

### 喂入文件

@context/ 下全部文件
@templates/ 下全部规范文件
@prd-docs/采购管理/采购入库单/（刚生成的 PRD 套件）
@prd-docs/库存管理/库存流水查询/（刚生成的 PRD 套件）
@prd-docs/库存管理/即时库存查询/（刚生成的 PRD 套件）
@prd-docs/采购管理/采购订单/（已有参照物 PRD 套件，作为对比基准）

### 提示词（直接复制）

```
请重新读取以下四个模块的 PRD 套件：
- prd-docs/采购管理/采购订单/（参照物——完整 PRD 套件）
- prd-docs/采购管理/采购入库单/（刚生成的）
- prd-docs/库存管理/库存流水查询/（刚生成的）
- prd-docs/库存管理/即时库存查询/（刚生成的）

对照 context/ 里的业务规则和 templates/ 里的规范，逐项检查以下维度：

1. **字段完整性**——字段清单是否覆盖了所有必须的字段？有没有该有但漏掉的？
2. **状态机闭环**（采购入库单）——状态流转是否有断点？是否有永远到不了的状态？执行层单据是否被错误地加了审核状态？
3. **规则一致性**——PRD 里写的业务规则是否和 context/07 的约束有矛盾？
4. **术语一致性**——是否混用了不同名称指代同一概念（对照 context/08 术语字典）？
5. **遗漏检查**——对照 context/06 的业务场景，PRD 是否覆盖了所有关键异常分支？
6. **模块间一致性**——采购入库单的字段清单和采购订单的字段清单，同一概念的定义是否一致？

列出所有发现的问题点，每条标注：
- 严重程度：高（阻断性错误）/ 中（功能遗漏或规则矛盾）/ 低（展示不一致）
- 所在模块和文件
- 建议修正方向

不直接修改文件。等我确认问题清单后再逐条修正。
```

### 讲师备注

**预期产出**：
- 问题清单，约 3-5 条
- 每条标注严重程度 + 模块 + 位置 + 修正建议

**点评要点**：
- 这是验证关卡——不过关不往下走前端
- 采购入库单状态机如果多了"已审核"状态——这是典型 AI 脑补
- 按什么标准查——标准来自 context/ 和 templates/，地基决定走查底线
- 先列清单再逐条修，不让 AI 直接改

## 后半部分：前端粗验 + 归档（第 8 步使用）

> 前端生成后，做终验并归档提示词。

### 喂入文件

@context/ 下全部文件
@templates/ 下全部规范文件
@front-prototype/src/（刚生成的全部前端代码）
@prd-docs/采购管理/采购订单/（采购订单 PRD 套件）

### 提示词 1：前端粗验

```
请重新读取采购订单的前端页面代码（front-prototype/src/pages/PurchaseOrder*.tsx）。

对照以下标准逐项检查：

1. **字段对齐**——页面上的字段名、状态枚举值是否与采购订单字段清单一致？
2. **交互合规**——表单中是否直接放了状态修改字段？（违反规范——状态变更必须通过按钮触发弹窗）
3. **审核锁定**——审核后的关键字段是否正确设为只读？（供应商、仓库、商品明细、单价）
4. **状态 Badge**——颜色是否按规范：草稿=灰、待审核=蓝、待入库=黄、部分入库=橙、已完成=绿、已作废=红？
5. **风格检查**——是 B 端管理后台风格（紧凑、信息密度高）还是展示站风格（大间距、少内容）？
6. **导入导出**——导入的校验规则是否完整（重复行判断、供应商/仓库/商品有效性）？导出的字段选择是否与字段清单对齐？

列出所有问题，标注严重程度和修正方向。
```

### 提示词 2：归档

```
帮我把今天对话中用到的以下几段提示词，整理成独立的 md 文件，保存到 prompts/ 目录：

1. prompts/context填充-业务场景.md —— 填充 context/06 的标准提示词模板
2. prompts/context填充-功能模块.md —— 填充 context/04 的标准提示词模板（强调从场景反推）
3. prompts/context填充-业务约束.md —— 填充 context/07 的标准提示词模板
4. prompts/PRD套件生成-业务单据.md —— 生成完整 PRD 套件（6 类产出物）的标准提示词模板
5. prompts/PRD套件生成-查询页面.md —— 生成查询型页面 PRD 套件的标准提示词模板
6. prompts/前端页面生成-并行多页面.md —— 多 Agent 并行生成前端页面的标准提示词模板

每个文件需要标注：适用场景、需要提前准备的文件、预期产出。
```
