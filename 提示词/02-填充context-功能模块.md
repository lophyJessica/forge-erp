## 提示词（直接复制）

```
请基于 context/01、02、03、05 的业务背景，以及 context/06 中梳理的业务场景，帮我填写 context/04-主要功能模块和清单.md。

核心思路：从业务场景反推系统需要哪些功能模块来支撑这些流转。context/06 走了"采购下单→审核→通知仓库→收货→入库→形成应付"的链路，自然反推出采购管理、库存管理、往来管理三个核心域。

要求：
1. 基于强盛科技的一期范围判断（最小可运行闭环），列出一期功能模块树。每个模块下列出核心功能点，用缩进表示层级。
2. 用 Mermaid 画一张模块间数据依赖关系图，标出关键的数据流向（采购订单→采购入库单→库存→应付，销售订单→销售出库单→库存→应收）。
3. 每个功能点标注一期/二期分界：🔴一期必做 / 🟡二期扩展。
4. 单独列一节"二期扩展模块"——哪些功能是一期故意不做的、在什么条件下可以扩展。

重要约束：
- 一期范围以 context/02 第"四、一期项目判断"为准
- 模块命名使用 context/05 中定义的术语（如"采购订单"不是"采购单"）
- 不要凭空添加 context 中没有的功能——比如不要写"供应商评级"，context/02 里没有
- 一期不做安全库存预警、不做自动补货建议——context/02 已明确
```

## 讲师备注

**预期产出**：
- 模块树：基础资料→采购管理→销售管理→零售管理→库存管理→往来管理→查询统计→系统设置
- Mermaid 依赖图体现核心数据流向
- 一期/二期边界标注清晰

**点评要点**：
- 强调顺序：先有 context/06 场景，再有 context/04 模块——从场景反推模块，而不是先拍模块再塞场景
- 展示 context/05 约束生效——AI 用了正确术语
- 如果 AI 写了"安全库存预警"——指出 context/02 明确说了一期不做，AI 凭通用知识脑补了
- 一期/二期分界 AI 只能给建议——最终裁定是 PM 做的
