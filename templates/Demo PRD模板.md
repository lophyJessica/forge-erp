# Demo PRD 模板

> 适用于：AI 生成前端原型前的页面级规格输入文档。
>
> 每个业务单据至少输出三份 Demo PRD：**列表页**、**新增/编辑页**、**详情页**。
> 本模板从「采购订单_Demo_列表页」「采购订单_Demo_新增编辑页」「采购订单_Demo_详情页」实际结构中提取。
>
> **核心原则**：Demo PRD 只写页面呈现规则（控件类型、交互行为、布局、文案），**严禁重复定义字段枚举值、计算规则、必填性**。这些定义以字段清单为唯一权威源。

---

## 通用文件头部

```markdown
# [单据名称]_Demo_[页面类型]

> **文档定位**：AI 生成 [页面名称] 前端原型的输入文档，聚焦页面结构、控件类型、交互行为。
> **字段定义**：字段枚举值、计算规则、必填性等见《[单据名称]字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.0 | YYYY-MM-DD
```

## 一、列表页 Demo PRD

### 1.1 页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | [模块名]列表 |
| 页面目标 | 一句话：这个列表帮用户完成什么任务 |
| 页面路径 | `/module/entities` |
| 骨架结构 | 查询卡片 + 列表卡片（两卡片结构） |

### 1.2 状态 Tab 栏（可选）

如果单据有状态且状态数量 ≥ 3，建议使用状态 Tab 栏作为列表页的主过滤入口。

| Tab 标签 | 过滤逻辑 |
| :--- | :--- |
| 全部 | 展示所有状态（默认选中） |
| [状态A] | 仅展示该状态 |

**Tab 交互规则**：
- 每个 Tab 右侧展示当前条件下的计数（格式：`Tab名 N`，超过 99 展示 `99+`，0 也展示）
- 切换 Tab 后列表刷新，页码重置，**不清空其他查询条件**
- 切换到具体状态 Tab 时，查询区的「状态」筛选条件隐藏（避免语义重叠），切换回「全部」时恢复

### 1.3 查询区

#### 布局说明
桌面端栅格布局（如 6 列），默认展示第一行条件，展开后显示第二行。右侧操作组：`展开/收起` → `重置` → `搜索`。

#### 查询条件

| 字段 | 控件类型 | 规格说明 |
| :--- | :--- | :--- |
| [字段名] | `BatchSearchInput` / `SelectWithSearch` / `Select` / `DateRange` / `Input` | 详细规格：展示格式、搜索方式、快捷选项、默认值 |

**控件类型速查**：
- `BatchSearchInput`：支持换行或逗号分隔多值，精确匹配
- `SelectWithSearch`：支持搜索的下拉选择，展示格式 `{Code} {Name}`
- `Select`：普通下拉选择，枚举值固定
- `DateRange`：日期区间选择器，含快捷选项（今天/本周/本月）

### 1.4 工具条

工具条位于查询区与表格之间。左侧放业务操作（如「新增」），右侧放表格级操作（如「导出」）。

| 按钮 | 位置 | 状态 | 说明 |
| :--- | :--- | :--- | :--- |
| 新增 | 左侧 | 常驻 | 跳转至新增页 |
| 导出 | 左侧 | 常驻 | 导出当前筛选结果 |

### 1.5 表格

#### 表格行为

| 行为 | 规格 |
| :--- | :--- |
| 行勾选 | 支持 / 不支持 |
| 列宽拖拽 | 支持 |
| 固定列 | 单据号列左固定；操作列右固定 |
| 分页 | 默认每页 20 条，支持切换（20/50/100） |
| 空状态 | 展示「暂无数据」插图 + 文字说明 |

#### 列定义

| 列名 | 内容说明 | 宽度 | 固定 |
| :--- | :--- | :--- | :--- |
| 勾选列 | 复选框 | 48px | 左固定 |
| [单据号] | 可点击进入详情页，链接色展示 | 180px | 左固定 |
| 状态 | 状态 Tag（按下表配色） | 100px | |
| ... | | | |
| 操作 | 行内文字操作（按下表动态展示） | 120px | 右固定 |

#### 状态 Tag 配色

| 状态 | 颜色语义 |
| :--- | :--- |
| 草稿 | 灰色 |
| 待审核 | 橙色 |
| 待执行 | 蓝色 |
| 部分完成 | 浅蓝色 |
| 已完成 | 绿色 |
| 已作废 | 红色 |

#### 行内操作

行内操作使用**文字动作**，按状态动态展示。**不渲染 disabled 操作**（不可用时不展示入口，不展示灰色禁用文字）。

| 状态 | 展示的操作 |
| :--- | :--- |
| [状态A] | 查看 / 编辑 / 提交审核 / 删除 |
| [状态B] | 查看 / 审核 / 驳回 / 作废 |

### 1.6 二次确认弹窗规格

| 触发操作 | 弹窗标题 | 弹窗内容 | 确认按钮文案 | 取消按钮文案 |
| :--- | :--- | :--- | :--- | :--- |
| 删除 | 确认删除 | 删除后不可恢复，确认删除？ | **确认删除** | 取消 |
| 作废 | 确认作废 | 作废后不可恢复，单据将进入已作废状态，确认作废？ | **确认作废** | 取消 |
| ... | | | | |

**弹窗文案要求**：
- 标题简洁（4-6 字），不重复操作名和字段名
- 内容必须告知操作后果（是否可恢复、影响范围）
- 确认按钮使用**危险语义**时，确认文案明确（如「确认删除」而非「确定」）

### 1.7 Mock 数据规格

#### 状态覆盖要求

| 状态 | 最少条数 | 备注 |
| :--- | :--- | :--- |
| [每个状态] | ≥ N | |
| **总计** | **≥ N** | |

#### 数据要求
- 展示格式示例
- 单号格式与连续递增说明
- 金额范围说明
- 所有数据使用虚构演示数据，不使用真实公司信息

### 1.8 反馈文案

| 操作 | 反馈类型 | 文案 |
| :--- | :--- | :--- |
| 操作成功 | Toast | 具体成功文案 |
| 操作失败 | Toast | 具体失败文案 |

## 二、新增/编辑页 Demo PRD

### 2.1 页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | [模块名]新增 / 编辑 |
| 页面目标 | 一句话说明页面任务 |
| 新增路径 | `/module/entities/new` |
| 编辑路径 | `/module/entities/{id}/edit` |
| 骨架结构 | 页头 → 表单区（头部信息卡片 + 明细卡片）→ 固定底部操作栏 |

### 2.2 固定底部操作栏

| 元素 | 规格 |
| :--- | :--- |
| 位置 | `fixed bottom-0 right-0` |
| 按钮顺序（从左到右） | `返回列表`（次操作）　`保存`（次操作）　`保存并提交`（主操作） |
| `保存` | 次要按钮样式（轻度边框） |
| `保存并提交` | 主要按钮样式（实心背景色） |

### 2.3 头部信息卡片

卡片标题：**基本信息**。布局：栅格（如 4 列桌面端）。

| 字段 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| [字段名] | `SelectWithSearch` / `DatePicker` / `Input` / `Textarea` / `NumberInput` | 单选/多选、数据来源、默认值、placeholder、字符限制、是否可清空 |

### 2.4 字段状态（编辑模式下区分草稿态 vs 审核后）

| 字段 | 草稿态 | 审核后 |
| :--- | :--- | :--- |
| [字段A] | 可编辑 | ReadonlyText 只读 |
| [字段B] | 可编辑 | 仍可编辑（审核后不锁定） |

### 2.5 商品/明细卡片

卡片标题：**商品明细**（或**明细清单**）。

#### 工具条
- 「选择商品」按钮 → 打开商品选择弹窗（支持搜索 + 多选）
- 「删除行」按钮 → 行内操作，删除对应行

#### 明细行列定义

| 列名 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| 序号 | ReadonlyText | 自动生成，从 1 起 |
| [商品编码] | ReadonlyText | 由商品选择弹窗写入，行内不可直接编辑 |
| [商品名称] | ReadonlyText | 选中商品后自动带出，快照只读 |
| [数量] | NumberInput | 正整数，最小值 1，最大值 9999999，不展示步进按钮 |
| [单价] | NumberInput | 最多 2 位小数，最小值 0，前缀 `¥` |
| [金额] | ReadonlyText | `数量 × 单价`，实时计算，千分位 |
| 行备注 | Input | 最多 100 字符，placeholder：`行备注（选填）` |
| 操作 | 文字按钮 | 「删除」 |

#### 明细行约束
- 至少保留 1 行，删除最后一行时提示「至少保留一行明细」
- 最多 N 行，超出时「选择商品」按钮 disabled 并提示

### 2.6 商品选择弹窗（如有）

说明调用哪个通用组件，以及本模块的调用参数配置。

## 三、详情页 Demo PRD

### 3.1 页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | [模块名]详情 |
| 页面目标 | 查看单据完整信息、状态时间线、关联下游单据 |
| 页面路径 | `/module/entities/{id}` |
| 骨架结构 | 状态区 → 基础信息卡片 → 明细卡片 → 关联单据卡片（如有）→ 固定底部操作栏 |

### 3.2 状态区

- 当前状态 Badge（颜色同列表页配色）
- 状态时间线（审核时间、入库时间、完成时间等）

### 3.3 基础信息卡片

所有头部字段只读展示，字段名: 字段值的 Key-Value 布局。

### 3.4 明细卡片

表格展示，列与列表页一致，只读。额外展示累计数量列（如累计已入库数量、未入库数量）。

### 3.5 关联单据卡片

列出下游单据列表，支持点击跳转。

### 3.6 底部操作按钮（按状态动态展示）

| 状态 | 展示的操作按钮 |
| :--- | :--- |
| [状态A] | 编辑 / 提交审核 / 删除 |
| [状态B] | 审核 / 驳回 / 作废 |
| [状态C] | 创建下游单据 / 关闭订单 |

## 四、通用交互规范（所有 Demo PRD 适用）

### 4.1 必填标识
- 必填字段标签前加红色星号 `*`
- 提交时校验，未填的必填字段标红并聚焦

### 4.2 错误反馈
- 保存/提交失败：Toast 错误提示，3 秒自动消失
- 阻断类校验：字段下方红色文字提示
- 网络异常：Toast「网络异常，请重试」

### 4.3 操作成功反馈
- 关键操作成功（提交、审核、作废）：Toast 成功提示
- 成功后自动跳回列表页，定位到操作前 Tab

### 4.4 字段联动规则
- 选中商品后自动带出名称、规格、单位 → 只读不可编辑
- 数量或单价变化 → 金额实时重算
- 客户选择 → 自动带出价格级别、账期、信用额度

### 4.5 并发保护
- 编辑页打开时记录数据版本
- 保存时对比版本，如已被他人修改则提示「数据已被他人修改，请刷新后重试」
