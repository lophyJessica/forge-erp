# 字段清单模板

> 本模板从「采购订单字段清单」实际结构中提取。字段清单是字段定义的**唯一权威源**（Single Source of Truth）。
>
> **引用规则**：下游文档（主PRD、Demo PRD）只引用字段名称写规则，严禁在此之外重复定义字段的枚举值、必填性、计算规则。
> **修改原则**：字段定义变化 → 只改本文件；下游文档由 AI 重新生成，不手动修改。

---

### 文件头部（必填）

```markdown
# [单据/模块名称]字段清单

> **文档定位**：[一句话说明本文件是哪些字段的唯一定义来源]
> **引用规则**：
> - 下游单据继承本单据字段时，标注「继承自 [来源单据名]」，不重复写字段定义
> - 下游文档只引用字段名称写规则，严禁重复定义字段本体
> **修改原则**：字段定义变化 → **只改本文件**；下游 Demo PRD 不手动修改，由 AI 重新生成
> **版本**：V1.0 | 更新时间：YYYY-MM-DD
```

## 一、头部字段（单据级）

> 每列的含义参见下方「列说明」。这个 11 列表格是本项目所有字段清单的标准格式。

| 字段名称 | 字段来源 | 取值说明 | 必填性 | 新增页 | 编辑页 | 列表展示 | 可筛选 | 详情展示 | 字段说明 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| | | | | | | | | | | |

### 列说明

| 列名 | 填写规范 |
| :--- | :--- |
| **字段名称** | 字段的中文名，全局统一。被下游引用时以此为准 |
| **字段来源** | 五选一：`系统生成`（自动创建，用户不可操作）/ `用户填写`（人工录入或选择）/ `系统计算`（基于其他字段实时计算）/ `系统带出并存储`（从档案自动带出，快照存储）/ `下游回写`（由下游单据确认后回写更新） |
| **取值说明** | 类型（文本/数字/日期/枚举）、格式（如 `YYYY-MM-DD`）、取值范围、枚举值列表。枚举值格式：`CODE=中文含义`，如 `DRAFT=草稿` |
| **必填性** | `必填` = 提交时必须填写，空则阻断；`选填` = 可为空；`系统自动` = 不展示给用户 |
| **新增页** | `可编辑` / `只读` / `不展示`。标注控件类型：`Input` / `Select` / `SelectWithSearch` / `DatePicker` / `NumberInput` / `Textarea` / `ReadonlyText` |
| **编辑页** | 同上。特别注意：审核后锁定字段标注为「可编辑（草稿态），审核后锁定时为只读」 |
| **列表展示** | `是` / `否`。列表展示的字段标注建议列宽（如 `180px`） |
| **可筛选** | `是` + 控件类型，如 `是（Input）` / `是（Select）` / `是（DateRange）` / `是（SelectWithSearch）`。不可筛选标注 `否` |
| **详情展示** | `是` / `否`。详情页固定置底的系统字段也可以标注 `是（置底）` |
| **字段说明** | 一句话解释字段的业务含义。如果涉及快照存储，写明「快照存储，与来源档案变化解耦」 |
| **备注** | 补充约束：审核后锁定规则、默认值规则、控件 placeholder、字符数限制、历史数据兼容说明 |

## 二、商品明细字段（行级）

> 格式与头部字段完全一致，11 列表格。

每行代表单据中的一种商品/一条明细。注意区分：
- **系统带出并存储**的字段（如商品名称、条码、规格）——选中商品后自动填充，快照存储
- **用户填写**的字段（如数量、单价）——需要人工录入
- **系统计算**的字段（如金额 = 数量 × 单价）——实时计算，不可编辑

---

## 三、统计汇总字段（单据级，系统计算）

> 格式与头部字段完全一致。

这些字段是对明细行的聚合计算：
- `商品种数 = 明细行数`
- `总数量 = 所有行数量之和`
- `总金额 = 所有行金额之和，保留 2 位小数`

## 五、展示与引用规则

- 被下游单据引用时的展示格式（如采购订单被入库单引用时展示 `PO{日期}-{序号}`）
- 列表页优先展示列的排序
- 审核后字段锁定规则（哪些字段审核后不可修改、哪些仍可修改）
- 快照存储字段清单（哪些字段是快照，来源档案修改后不影响已有单据）
- 数量/金额字段的展示格式（千分位、小数位数、前缀符号）
