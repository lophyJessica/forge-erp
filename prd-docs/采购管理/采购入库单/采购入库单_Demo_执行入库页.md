# 采购入库单_Demo_执行入库页

> **文档定位**：AI 生成采购入库单执行入库页（新增/编辑页）前端原型的输入文档，聚焦页面结构、控件类型、交互行为。
> **字段定义**：字段枚举值、计算规则、必填性等见《采购入库单字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.0 | 2026-07-04

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 执行采购入库 |
| 页面目标 | 仓管员在此页面录入到货清点的实收数量与合格入库数量，点击确认正式办理入库 |
| 新增路径 | `/purchase/receipts/new?source_id={po_id}` (PO下推跳转) |
| 编辑路径 | `/purchase/receipts/{id}/edit` (草稿态PI编辑) |
| 骨架结构 | 页头 $\rightarrow$ 表单区（基本信息卡片 + 商品明细卡片） $\rightarrow$ 固定底部操作栏 |

---

## 二、固定底部操作栏

| 元素 | 规格 |
| :--- | :--- |
| 位置 | `fixed bottom-0 right-0` |
| 按钮顺序（从左到右） | `返回列表`（次要操作，轻度边框）　`保存草稿`（次要操作，轻度边框）　`确认入库`（主要操作，实心背景） |
| **返回列表** | 点击后若表单已被修改，弹出二次确认「数据尚未保存，确定返回列表？」 |
| **保存草稿** | 点击后进行基础格式校验，保存成功后留在当前页，并弹出 Toast 提示「草稿已保存」 |
| **确认入库** | 点击后执行全量校验（含超收强阻断），校验通过后弹出二次确认「确认后库存正式生效，确认入库？」 |

## 三、基本信息卡片

卡片标题：**基本信息**。布局：桌面端 4 列栅格。

| 字段 | 页面呈现规则 | 备注 |
| :--- | :--- | :--- |
| 采购入库单号 | 展示为 `ReadonlyText`。若为新建页，显示为 `系统自动生成` | 确认后系统生成单号 |
| 来源采购单号 | 展示为 `ReadonlyText`。链接色，可点击新窗口打开对应 PO 详情 | 继承自 PO，不可改 |
| 供应商 | 展示为 `ReadonlyText`。格式：`{Code} {Name}` | 继承自 PO，不可改 |
| 入库仓库 | 展示为 `ReadonlyText`。格式：`{Code} {Name}` | 继承自 PO，不可改 |
| 入库日期 | 控件类型：`DatePicker`。可清除；默认回填当前系统日期 | 确认入库后锁定为只读 |
| 入库备注 | 控件类型：`Textarea`。字符上限 200 字，提示：`请输入入库备注（选填）` | 确认入库后锁定为只读 |

## 四、商品明细卡片

卡片标题：**入库明细清单**。

### 4.1 明细工具条

- **无“选择商品”按钮**：PI 明细行必须完全来源于 PO 明细，禁止手动添加其他商品。
- **行“删除”按钮**：支持行内删除。当订单内某商品本次完全没有到货时，仓管员可将该行从本次 PI 单据中删除（仅代表本次入库不含该商品，PO 尚未入库的余额依然会保留在 PO 中，不受影响）。

### 4.2 表格列定义（从左到右）

| 列名 | 呈现规格与控件类型 | 详细交互说明 |
| :--- | :--- | :--- |
| 序号 | `ReadonlyText` | 自动生成，从 1 起递增 |
| 商品编码 | `ReadonlyText` | 继承只读 |
| 商品名称 | `ReadonlyText` | 快照只读 |
| 商品条码 / 规格 | `ReadonlyText` | 快照只读，合并单列展示以提高表格信息密度 |
| 单位 | `ReadonlyText` | 快照只读 |
| 订单数量 | `ReadonlyText` | 仅做信息参考，只读 |
| 订单未入库数量 | `ReadonlyText` | 本次实收和入库的核心上限校验基准，高亮字展示 |
| **实收数量** | `NumberInput` | **必填，正整数**。由用户输入。默认回填为“未入库数量”。当修改实收数时，若“入库数量”未填写，则自动将“入库数量”同步回填为等值。 |
| **入库数量** | `NumberInput` | **必填，正整数**。由用户输入。默认回填为“未入库数量”。修改时实时触发金额重算。 |
| 采购单价 | `ReadonlyText` | 继承 PO 的含税单价，格式：`¥{价格}` |
| 金额（含税） | `ReadonlyText` | 系统实时计算并展示：`入库数量 × 采购单价`。千分位，2 位小数 |
| 行备注 | `Input` | 字符上限 100 字。提示：`行备注（选填）` |
| 操作 | 文字按钮 | **删除**。点击后从本次 PI 中移除该商品行 |

### 4.3 列表底部汇总信息

明细表格右下方展示统计汇总，为只读计算：
- 商品种数：`明细行数`
- 入库总数量：`所有行“入库数量”之和`
- 入库总金额：`所有行“金额（含税）”之和`，展示格式如 `¥26,500.00`

## 五、页面字段联动与控制规则

### 5.1 智能回填联动

1.  **实收联动入库**：仓管员在明细行输入 `实收数量`（例如输入 8）后，若对应的 `入库数量` 仍为 0 或空，系统自动将 `入库数量` 同步修改为 8，以减少仓管员重复输入的次数。
2.  **金额实时重算**：一旦明细行 `入库数量` 发生变化，系统自动通过公式：`金额 = 入库数量 × 采购单价` 实时重新计算本行金额，并累加计算更新底部的“入库总金额”和“入库总数量”。

### 5.2 强控拦截交互控制

1.  **超收警告**：当用户在某行填入的 `实收数量` 或 `入库数量` 大于 `订单未入库数量` 时，系统在输入框下方立即展示红色提示语「不能超过订单未入库数({未入库数})」，并将该输入框标红。此时底部“确认入库”按钮变更为置灰不可点击状态。
2.  **收发不符校验**：当用户在某行填入的 `入库数量` 大于 `实收数量` 时，系统在入库数量输入框下方立即提示「入库数不能大于实收数」，标红并置灰“确认入库”按钮。
3.  **明细为空校验**：若仓管员删除了全部商品行，明细卡片展示「暂无入库商品，请重新从采购订单下推」，并置灰“确认入库”和“保存草稿”按钮。
