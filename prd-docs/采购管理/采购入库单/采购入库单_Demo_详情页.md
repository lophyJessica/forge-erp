# 采购入库单_Demo_详情页

> **文档定位**：AI 生成采购入库单详情页前端原型的输入文档，聚焦页面结构、展示布局和动态操作按钮。
> **字段定义**：字段枚举值、计算规则、必填性等见《采购入库单字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.0 | 2026-07-04

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 采购入库单详情 |
| 页面目标 | 提供对已确认、草稿及已作废采购入库单的完整信息查看、下游溯源和流程流转按钮 |
| 页面路径 | `/purchase/receipts/{id}` |
| 骨架结构 | 页头(单号+状态) $\rightarrow$ 状态区 $\rightarrow$ 基本信息卡片 $\rightarrow$ 明细清单卡片 $\rightarrow$ 关联下游卡片 $\rightarrow$ 固定底部操作栏 |

---

## 二、状态区

状态区位于页面最上方，用以反映该张单据当前的生命周期节点。

### 2.1 状态 Badge
- 页面左上角紧邻单据号展示状态 Badge，配色与列表页保持一致：
  - 草稿：灰色 Badge
  - 已确认：绿色 Badge
  - 已作废：红色 Badge

### 2.2 状态时间线
以水平时间轴展示单据的关键生命周期节点，只读不可编辑：
*   **创建时间**：`创建人` 于 `创建时间` 从 PO 下推创建
*   **作废时间**（仅在状态=已作废时展示）：`操作人` 于 `最后修改时间` 手动作废
*   **入库生效时间**（仅在状态=已确认时展示）：`确认人` 于 `确认时间` 确认入库，库存及应付款已生效

---

## 三、基本信息卡片

卡片标题：**基本信息**。采用只读 Key-Value（字段名: 字段值）的栅格化布局展示（桌面端 4 列）。

- **供应商**：`{供应商编码} {供应商名称}`
- **入库仓库**：`{仓库编码} {仓库名称}`
- **入库日期**：`YYYY-MM-DD`
- **来源采购单号**：`PO{日期}-{序号}`，链接格式，点击在新窗口打开关联的采购订单详情
- **采购备注**：直接引用采购订单备注展示，无值时显示 `-`
- **入库备注**：展示本单入库备注，无值时显示 `-`

---

## 四、明细清单卡片

卡片标题：**入库明细清单**。

### 4.1 表格列定义（只读）

| 列名 | 内容说明 | 宽度 |
| :--- | :--- | :--- |
| 序号 | 自动生成，从 1 起递增 | 60px |
| 商品编码 | 展示商品编码，只读 | 120px |
| 商品名称 | 商品名称快照，只读 | 200px |
| 条码 / 规格 | 商品条码与规格，只读 | 180px |
| 单位 | 商品计量单位快照 | 80px |
| 订单数量 | 对应 PO 商品行的计划采购数 | 100px |
| 未入库数量 | 本单生成时刻的未入库剩余量 | 120px |
| 实收数量 | 实际点数到货量 | 100px |
| **入库数量** | 正式验收入库数量 | 100px |
| 采购单价 | 采购含税单价，格式：`¥{价格}` | 110px |
| 金额（含税） | 本行入库含税金额：`入库数量 × 采购单价`，格式如 `¥26,500.00` | 130px |
| 行备注 | 本行商品的特别备注，无值显示 `-` | 150px |

### 4.2 底部统计汇总（只读）
- 商品种数：`N 种`
- 入库总数量：`N`
- 入库总金额：`¥{总金额}`（千分位，2 位小数，大号加粗字体展示）

## 五、关联下游卡片

> 仅在入库单状态为 **已确认** 时展示，以实现单据全链路流转的闭环跟踪。

*   **关联库存流水**：展示自动生成的流水号 `FL{日期}-{序号}`，点击可跳转查看该库存流水的出入库台账明细。
*   **关联应付款记录**：展示自动生成的应付款记录单号 `AP{日期}-{序号}`，显示含税总金额、核销状态（未核销/部分核销/已核销），点击可跳转财务往来页面。
*   **关联采购退货单**（如有）：若本入库单已发起退货，在此展示退货单号 `PR{日期}-{序号}` 及其状态，支持多笔退货单关联（1:N），可点击跳转。若未发起退货，本项不展示。

---

## 六、固定底部操作栏

底部操作栏位于页面最下方，按钮根据单据状态进行动态显隐：

| 当前状态 | 展示的操作按钮（从左到右） | 交互行为与跳转 |
| :--- | :--- | :--- |
| **草稿** | `返回列表` (次)　`删除` (次)　`作废` (次)　`编辑` (次)　`确认入库` (主) | - **删除/作废**：触发二次确认弹窗，确认后Toast提示并自动跳转回列表页<br/>- **编辑**：点击跳转至《采购入库单执行入库页》的编辑模式<br/>- **确认入库**：触发全面校验与二次确认，确认后Toast提示并刷新当前页（状态变为已确认） |
| **已确认** | `返回列表` (次)　`创建退货单` (主) | - **创建退货单**：点击后下推创建采购退货单（PR），自动跳转至退货单新增页，并继承本单商品和价格快照 |
| **已作废** | `返回列表` (次) | - **返回列表**：点击跳转回入库单列表页 |
