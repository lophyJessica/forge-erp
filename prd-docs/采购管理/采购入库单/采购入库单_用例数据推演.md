## 3. 推演案例

### 案例 1：分批到货伴随破损拒收（采购10件 $\rightarrow$ 到8件 $\rightarrow$ 破损退2件 $\rightarrow$ 实入6件）

> **目标**：验证在到货量与计划量不符，且实收与入库不一致时的多口径数量回写、库存更新及财务计账准确性。

**场景**：
采购订单计划采购数量为 10 个，单价 50 元。供应商送货 8 个（包含 2 个破损当场拒收），实际验收入库 6 个。

**初始业务单据数据**：

*   **关联采购订单**：`PO20260704-0001`
    *   状态：已审核 (`PENDING_STOCK_IN`)
    *   明细行：SKU001，采购数量 = 10，累计已入库数量 = 0，未入库数量 = 10，含税单价 = 50.00，金额 = 500.00

**推演步骤**：

| 步骤 | 操作人/系统 | 操作 | 数据变化（字段 = 旧值 → 新值） | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 仓管员 | 从 PO20260704-0001 下推创建入库单 | `PI单号 = PI20260704-0001（新生成）`<br>`单据状态：（无）→ 草稿`<br>`商品编码：继承 = SKU001`<br>`未入库数量：继承 = 10`<br>`实收数量：0`<br>`入库数量：0` | 单号格式校验合格；来源继承无遗漏；实收/入库数默认为0。 |
| 2 | 仓管员 | 录入点数及验货结果 | `实收数量：0 → 8`<br>`入库数量：0 → 6`<br>`金额（含税）：0.00 → 300.00` | 行金额公式重算：`6 × 50 = 300.00`，保留 2 位小数。 |
| 3 | 仓管员 | 点击“确认入库”按钮 | `入库状态：草稿 → 已确认`<br>`确认人 = 仓管员_张三（新记录）`<br>`确认时间 = 2026-07-04 14:30:15（新记录）`<br>系统对 PI 进行锁定（全部字段只读） | 拦截规则校验：实入 6 $\le$ 实收 8 $\le$ 未入 10，校验通过。进入只读终态。 |
| 4 | 系统自动 | 增加仓库商品现存量并生成流水 | `WH001 SKU001现存库存：15 → 21`<br>`可用库存：15 → 21`<br>`库存流水号 = FL20260704-00000001（新生成）`<br>`流水类型 = 采购入库`<br>`流水数量 = +6` | 库存扣减/累加正确。流水类型和数量匹配。可用库存等额增加。 |
| 5 | 系统自动 | 回写采购订单 PO 状态及数量 | `PO商品行累计已入库数量：0 → 6`<br>`PO商品行未入库数量：10 → 4`<br>`PO单据头状态：已审核 → 部分入库` | 累计数累加而非覆盖。未入库数变更为 4。状态正常跳转至“部分入库”。 |
| 6 | 系统自动 | 推送生成财务应付款明细记录 | `应付款流水 = AP20260704-001（新生成）`<br>`关联供应商 = S001`<br>`应付总金额（含税）：0.00 → 300.00`<br>`核销状态：未核销` | 应付款自动生成；金额与 PI 入库总金额一致。 |

## 4. 金额与税额精确验算

按照进销存财务标准，使用**税额差额法**对本次入库的 6 件商品进行财务明细金额核算（税率 13%）：

| 项目 | 公式 / 计算过程 | 结果值 | 精度要求 |
| :--- | :--- | :--- | :--- |
| **入库数量** | 实物点数合格量 | **6** | 整数 |
| **含税单价** | 继承 PO | **50.00 元** | 2 位小数 |
| **价税合计** | $数量 \times 含税单价 = 6 \times 50.00$ | **300.00 元** | 2 位小数 |
| **不含税单价** | $含税单价 \div (1 + 税率) = 50.00 \div 1.13$ | **44.247788 元** | 6 位小数，不四舍五入保留完整精度 |
| **金额（不含税）** | $不含税单价 \times 数量 = 44.247788 \times 6 = 265.486728$ | **265.49 元** | 2 位小数，四舍五入 |
| **税额** | $价税合计 - 金额（不含税） = 300.00 - 265.49$ | **34.51 元** | 2 位小数（差额法，不对税额进行单独舍入） |

*   **校验结论**：
    *   $金额（265.49元） + 税额（34.51元） = 价税合计（300.00元）$，账目绝对平齐，无尾差漏洞。

## 6. 设计走查清单

*   [x] **单号规范**：生成入库单号为 `PI20260704-0001`，格式吻合《单据分层与编码规则》。
*   [x] **三数量独立**：明细行上，订单计划（10件）、实收（8件）、入库（6件）分开记录且满足 $6 \le 8 \le 10$。
*   [x] **未入库递减**：PI 确认后，PO 未入库数减少为 4（支持后续继续到货），累计入库增加为 6，是累加而非覆盖。
*   [x] **库存同步更新**：仓库实物现存增加 6，可用库存同步增加 6（由于采购阶段无占用库存，可用库存与现存保持同步增加）。
*   [x] **税额尾差核算**：价税合计 300.00 元，不含税金额 265.49 元，税金 34.51 元，符合“税额差额法”且加和无误差。
*   [x] **应付记录触发**：在财务往来中自动生成了 300.00 元未核销应付款。
*   [x] **终态锁定**：PI 确认后变更为“已确认”，全字段只读锁死，不可修改和反确认。
