## 1. 推演目标

- 验证采购退货单（PR）从已确认采购入库单（PI）下推、编辑、确认、下推 PRO 的数据闭环。
- 重点关注：来源继承、可退货数量、退货金额、状态流转、确认后锁定、PR 不直接影响库存和应付。
- 不验证：PRO 确认出库后的库存扣减与应付冲减细节，该部分属于采购退货出库单范围。

## 3. 推演案例

### 案例 1：正常流程（部分退货申请）

**场景**：PI 已确认入库 SKU001 10 个、SKU002 20 个。复检发现部分商品有问题，发起 PR 退 SKU001 3 个、SKU002 5 个。

| 步骤 | 操作人/系统 | 操作 | 数据变化 | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 采购员 | 从 PI20260704-0001 下推创建 PR | `PR单号 = PR20260707-0001（新生成）`<br>`退货状态：（无）→ 草稿`<br>`来源入库单号 = PI20260704-0001（继承）`<br>`供应商 = S001 深圳强盛电子（继承）` | 来源必须是已确认 PI；字段继承正确 |
| 2 | 采购员 | 编辑退货数量和原因 | `SKU001本次退货数量：10 → 3`<br>`SKU002本次退货数量：20 → 5`<br>`退货原因：空 → 复检发现针脚歪斜和外壳刮伤` | 数量可改小；原因必填 |
| 3 | 系统 | 实时计算金额和汇总 | `SKU001退货金额：500.00 → 150.00`<br>`SKU002退货金额：240.00 → 60.00`<br>`退货总数量：30 → 8`<br>`退货总金额：740.00 → 210.00` | 金额和汇总公式正确 |
| 4 | 采购员 | 确认退货 | `退货状态：草稿 → 已确认`<br>`确认人 = 采购员_李四`<br>`确认时间 = 2026-07-07 10:20:30` | 状态闭环；确认后锁定 |
| 5 | 系统 | 更新 PR 申请口径 | `SKU001已退货申请数量：0 → 3`<br>`SKU001可退货数量：10 → 7`<br>`SKU002已退货申请数量：0 → 5`<br>`SKU002可退货数量：20 → 15` | 已确认 PR 计入可退货余额 |
| 6 | 仓管员 | 下推 PRO | `PRO单号 = PRO20260707-0001（新生成）`<br>`PRO来源PR = PR20260707-0001` | PR 已确认才可下推 PRO |

**数字校验**：

| 项目 | 公式 | 结果 |
| :--- | :--- | :---: |
| SKU001 退货金额 | `3 × 50.00` | 150.00 |
| SKU002 退货金额 | `5 × 12.00` | 60.00 |
| 退货总数量 | `3 + 5` | 8 |
| 退货总金额 | `150.00 + 60.00` | 210.00 |
| SKU001 剩余可退 | `10 - 3` | 7 |
| SKU002 剩余可退 | `20 - 5` | 15 |

**状态快照**：

| 时间点 | PR 状态 | SKU001 可退货数量 | SKU002 可退货数量 | SKU001 现存 | SKU002 现存 | 应付影响 |
| :--- | :--- | :---: | :---: | :---: | :---: | :--- |
| 初始 | - | 10 | 20 | 25 | 40 | PI 已形成 740.00 应付 |
| PR 草稿 | 草稿 | 10 | 20 | 25 | 40 | 无变化 |
| PR 已确认 | 已确认 | 7 | 15 | 25 | 40 | 无变化 |
| PRO 草稿 | 已确认 | 7 | 15 | 25 | 40 | 无变化 |

**走查清单**：
- [x] PR 单号符合 `PR{YYYYMMDD}-{4位序号}`。
- [x] PR 继承 PI 的供应商、仓库、商品、单价。
- [x] 退货原因必填，确认后不可修改。
- [x] PR 确认后不扣减现存、不生成 FL、不冲减应付。
- [x] 已确认 PR 数量计入同一 PI 行的可退货余额。

**结论**：通过。

### 案例 2：分批退货

**场景**：案例 1 已确认退货 SKU001 3 个后，又发现 SKU001 还有 4 个质量问题，从同一 PI 再发起第二张 PR。

| 步骤 | 操作人/系统 | 操作 | 数据变化 | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 采购员 | 再次从 PI 下推 PR | `PR单号 = PR20260707-0002（新生成）`<br>`SKU001可退货数量 = 7` | 可退货数量应扣除第一张已确认 PR |
| 2 | 采购员 | 填写 SKU001 本次退货 4 | `SKU001本次退货数量：7 → 4`<br>`SKU001退货金额：350.00 → 200.00` | `4 × 50.00 = 200.00` |
| 3 | 采购员 | 确认退货 | `PR20260707-0002状态：草稿 → 已确认`<br>`SKU001已退货申请数量：3 → 7`<br>`SKU001可退货数量：7 → 3` | 多张 PR 累加，不覆盖 |

**数字校验**：SKU001 已确认 PR 累计 `3 + 4 = 7`；剩余可退 `10 - 7 = 3`；第二张 PR 金额 `4 × 50.00 = 200.00`。

**结论**：通过。

### 案例 3：草稿删除/作废

| 步骤 | 操作人/系统 | 操作 | 数据变化 | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 采购员 | 下推生成 PR 草稿 | `PR单号 = PR20260707-0003（新生成）`<br>`状态：（无）→ 草稿` | 可进入编辑 |
| 2A | 采购员 | 删除草稿 | `PR20260707-0003：草稿 → 物理消失` | 删除仅草稿可用；不影响 PI |
| 2B | 采购员 | 或选择作废草稿 | `PR20260707-0003：草稿 → 已作废` | 作废保留记录，只读 |

| 处理方式 | PR 是否可查 | PI 可退货数量 | 库存 | 应付 |
| :--- | :---: | :---: | :---: | :---: |
| 删除 | 否 | 不变 | 不变 | 不变 |
| 作废 | 是 | 不变 | 不变 | 不变 |

**结论**：通过。

### 案例 4：超量退货阻断

**场景**：SKU001 原入库 10 个，已有两张已确认 PR 累计退货 7 个，第三张 PR 尝试退 4 个。

| 步骤 | 操作人/系统 | 操作 | 数据变化 | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 系统 | 计算当前可退货数量 | `SKU001可退货数量 = 10 - 7 = 3` | 可退货口径正确 |
| 2 | 采购员 | 输入本次退货数量 4 | `本次退货数量：3 → 4` | 草稿可输入，但保存/确认需校验 |
| 3 | 系统 | 确认时阻断 | `PR状态：草稿 → 草稿（不变）`<br>`已退货申请数量：7 → 7（不变）` | 不允许超退 |

**预期提示**：`商品 华强北特种接插件 可退货数量不足，当前最多可退 3 件`

**结论**：通过。

### 案例 5：边界与并发

| 场景 | 操作 | 预期系统行为 |
| :--- | :--- | :--- |
| 来源 PI 为草稿 | 下推 PR | 阻断，提示仅已确认 PI 可发起退货 |
| 退货原因为空 | 确认 PR | 阻断，提示填写退货原因 |
| 明细行为空 | 确认 PR | 阻断，提示至少保留一行退货商品 |
| 已确认 PR 尝试编辑 | 点击编辑 | 不展示编辑入口；直接访问编辑地址也阻断 |
| 两人同时确认同一 PI 行退货 | 后提交者确认 | 系统重新计算可退货数量，超出则阻断 |
| 已确认 PR 重复下推 PRO | 第二次下推 | 阻断，提示已存在采购退货出库单 |
