# 采购退货单_Demo_新增编辑页

> **状态**：已补齐
> **角色**：Demo·新增编辑页　|　类型：业务单据
> **权威层级**：采购退货单主PRD > 字段清单 > 业务规则 > 本文件
> **参照套件**：页面结构参考《采购订单_Demo_新增编辑页》
> **版本**：V1.0 | 2026-07-07

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 采购退货单新增 / 编辑 |
| 页面目标 | 基于已确认 PI 发起退货申请，录入退货数量和退货原因 |
| 新增页路径 | `/purchase/returns/new?source_id={pi_id}` |
| 编辑页路径 | `/purchase/returns/{id}/edit` |
| 骨架结构 | 页头 → 来源信息卡片 → 退货信息卡片 → 退货明细卡片 → 固定底部操作栏 |

新增页必须带 `source_id`，不支持空白新增。

---

## 二、页头

| 元素 | 规格 |
| :--- | :--- |
| 标题（新增模式） | `发起采购退货` |
| 标题（编辑模式） | `编辑采购退货单 {采购退货单号}` |
| 副标题 | 展示来源入库单号；可点击跳转 PI 详情 |

---

## 三、固定底部操作栏

| 元素 | 规格 |
| :--- | :--- |
| 位置 | `fixed bottom-0 right-0`，内边距 `px-6 py-3` |
| 按钮顺序 | `返回列表` / `保存草稿` / `确认退货` |
| 返回列表 | 文字按钮 |
| 保存草稿 | 次要按钮 |
| 确认退货 | 主按钮 |

若当前 PR 不是草稿态，页面不允许进入编辑模式；直接访问编辑地址时跳转详情页并提示「当前状态不可编辑」。

---

## 四、来源信息卡片

卡片标题：**来源信息**。全部字段为只读展示。

| 字段 | 控件类型 | 展示规格 |
| :--- | :--- | :--- |
| 来源入库单号 | `ReadonlyLink` | 展示 PI 单号，点击跳转采购入库单详情页 |
| 来源采购单号 | `ReadonlyLink` | 展示 PO 单号，点击跳转采购订单详情页 |
| 供应商 | `ReadonlyText` | `{Code} {Name}` |
| 退货仓库 | `ReadonlyText` | `{Code} {Name}` |

---

## 五、退货信息卡片

卡片标题：**退货信息**。桌面端 4 列栅格。

| 字段 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| 退货日期 | `DatePicker` | 默认今天；不可选择未来日期；不可清空 |
| 退货原因 | `Textarea` | 必填；最多 200 字；默认 3 行；placeholder：`请输入退货原因` |
| 退货备注 | `Textarea` | 选填；最多 200 字；默认 3 行；placeholder：`请输入退货备注（选填）` |

---

## 六、退货明细卡片

卡片标题：**退货明细**。明细来自 PI，可在草稿态删除不退的行或改小退货数量，不提供商品选择弹窗。

### 6.1 表格操作

| 操作 | 位置 | 说明 |
| :--- | :--- | :--- |
| 删除行 | 行内末尾 | 删除不需要退货的商品行，无需二次确认 |
| 恢复来源行 | 工具条右侧 | 打开“已删除来源行”弹窗，可恢复本次编辑中删除的 PI 行 |

至少保留 1 行，删除最后一行时提示「至少保留一行退货商品」；不允许新增非来源 PI 的商品行。

### 6.2 明细行列定义

| 列名 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| 序号 | `ReadonlyText` | 从 1 开始 |
| 商品编码 | `ReadonlyText` | 继承 PI |
| 商品名称 | `ReadonlyText` | 继承 PI 快照 |
| 商品条码 | `ReadonlyText` | 无值展示 `-` |
| 规格型号 | `ReadonlyText` | 无值展示 `-` |
| 单位 | `ReadonlyText` | 继承 PI |
| 原入库数量 | `ReadonlyText` | 整数 |
| 已退货申请数量 | `ReadonlyText` | 整数 |
| 可退货数量 | `ReadonlyText` | 整数 |
| 本次退货数量 | `NumberInput` | 正整数；最大值由字段清单与业务规则定义；不展示步进按钮 |
| 单价（含税） | `ReadonlyText` | `¥N.NN` |
| 退货金额（含税） | `ReadonlyText` | 随本次退货数量实时变化，千分位展示 |
| 行备注 | `Input` | 最多 100 字；placeholder：`行备注（选填）` |
| 操作 | 文字按钮 | `删除` |

### 6.3 汇总行

| 汇总项 | 展示方式 |
| :--- | :--- |
| 商品种数 | `共 N 种商品` |
| 退货总数量 | `合计 N 件` |
| 退货总金额 | `¥ N（千分位，2 位小数）` |

## 七、校验规则

Demo 只描述触发和呈现方式；字段定义和规则以《采购退货单字段清单》《采购退货单_业务规则规格》为准。

| 校验对象 | 触发时机 | 错误呈现 |
| :--- | :--- | :--- |
| source_id 缺失或无效 | 页面加载 | 展示错误页文案「缺少来源采购入库单，无法发起采购退货」 |
| 退货日期 | 失焦 + 保存/确认 | 字段下方红字 |
| 退货原因 | 点击确认退货 | 字段下方红字：`请填写退货原因` |
| 明细为空 | 保存/确认 | Toast：`明细行不能为空，请至少保留一行退货商品` |
| 本次退货数量 | 值变更失焦 + 保存/确认 | 行内红字，确认时 Toast 汇总第一条错误 |
| 并发导致可退货数量变化 | 确认退货 | 弹窗提示刷新数据，并标红冲突行 |

---

## 八、操作行为说明

| 操作 | 行为 |
| :--- | :--- |
| 页面加载 | 根据 `source_id` 拉取 PI 快照和可退货数量；若无可退货行，阻断进入 |
| 保存草稿 | 执行基础格式校验；保存成功后留在当前页 |
| 确认退货 | 先弹二次确认；确认后执行完整校验；成功后跳转详情页 |
| 返回列表 | 有未保存改动时弹窗确认「当前有未保存的内容，确认离开？」 |
| 删除行 | 从本次 PR 明细移除该行；不影响来源 PI |
| 恢复来源行 | 仅恢复本次编辑中删除且仍有可退货数量的来源行 |

---

## 九、模式差异汇总

| 差异项 | 新增模式 | 编辑模式（草稿） |
| :--- | :--- | :--- |
| 页面标题 | 发起采购退货 | 编辑采购退货单 {采购退货单号} |
| 数据来源 | 根据 source_id 初始化 | 回显已保存 PR 草稿 |
| 单号 | 保存后生成 | 页头展示 |
| 可编辑字段 | 退货日期、退货原因、退货备注、本次退货数量、行备注 | 同新增模式 |
| 可进入状态 | - | 仅草稿 |

## 十、反馈文案

| 操作 | 反馈类型 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 确认退货成功 | Toast | 采购退货单已确认 |
| 离开未保存 | Modal | 当前有未保存的内容，确认离开？ |
| 至少保留一行 | Toast | 至少保留一行退货商品 |
| 来源无效 | Error | 缺少来源采购入库单，无法发起采购退货 |
