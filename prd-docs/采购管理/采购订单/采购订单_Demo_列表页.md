## 二、状态 Tab 栏

### 2.1 Tab 项定义

状态 Tab 栏位于列表卡片顶部、查询区上方，是列表页的主要状态过滤入口。

| Tab 标签 | 过滤逻辑 |
| :--- | :--- |
| 全部 | 展示所有状态的订单（默认选中） |
| 草稿 | 仅展示「草稿」状态 |
| 待审核 | 仅展示「待审核」状态 |
| 待入库 | 仅展示「待入库」状态 |
| 部分入库 | 仅展示「部分入库」状态 |
| 已完成 | 仅展示「已完成」状态 |
| 已作废 | 仅展示「已作废」状态 |

### 2.2 计数规格

- 每个 Tab 标签右侧展示当前条件下的订单数量，格式：`Tab名称 N`（如 `草稿 3`）
- 计数超过 99 时展示 `99+`
- 计数为 0 时仍展示（展示 `0`，不隐藏 Tab）
- 计数反映**全部查询条件叠加后**的结果数（Tab 本身的状态过滤 + 查询区其他筛选条件）
- 切换 Tab 或查询条件变更后，计数同步刷新

### 2.3 交互行为

- 点击 Tab 切换后，列表数据及分页立即刷新，页码重置为第 1 页
- 切换到具体状态 Tab（非「全部」）时，查询区「订单状态」筛选条件**隐藏**，避免与 Tab 语义重叠
- 切换回「全部」Tab 时，「订单状态」筛选条件**恢复展示**
- Tab 切换不清空其他查询条件（供应商、日期等保留）

## 三、查询区

### 3.1 布局说明

- 桌面端 6 列栅格，默认展示 1 行（6 个条件）；展开后展示第 2 行（剩余条件）；批量搜索、日期范围条件均单列宽度
- 右侧操作组：`展开/收起` → `重置` → `搜索`

### 3.2 查询条件

| 字段 | 控件类型 | 规格说明 |
| :--- | :--- | :--- |
| 采购单号 | `BatchSearchInput` | 支持换行或英文逗号分隔多值，精确匹配；最多 1000 行 |
| 供应商 | `SelectWithSearch` | 展示格式 `{Code} {Name}`；支持编码或名称搜索；单选；可清空 |
| 入库仓库 | `SelectWithSearch` | 展示格式 `{Code} {Name}`；支持编码或名称搜索；单选；可清空 |
| 订单状态 | `Select` | 枚举单选，见字段清单；含「全部」选项，默认「全部」；不支持搜索；**仅在「全部」Tab 下展示，切换至具体状态 Tab 后自动隐藏** |
| 下单日期 | `DateRange` | 精确到日；可只选一端；开始不晚于结束；含「今天」「本周」「本月」快捷选项 |
| 预计到货日期 | `DateRange` | 精确到日；可只选一端 |
| 最后修改时间 | `DateRange` | 精确到日；展开后展示 |

> 前 6 个条件默认展示，第 7 个「最后修改时间」在展开状态下展示。

工具条位于查询卡片与表格之间，左侧放业务操作，右侧放表格级操作。

| 按钮 | 位置 | 状态 | 说明 |
| :--- | :--- | :--- | :--- |

## 五、表格

### 5.1 表格行为

| 行为 | 规格 |
| :--- | :--- |
| 行勾选 | 支持，全选/单选；勾选键为单据 id |
| 列宽拖拽 | 支持 |
| 固定列 | 采购单号列左固定；操作列右固定 |
| 分页 | 默认每页 20 条，支持切换每页数量（20/50/100） |
| 空状态 | 展示「暂无数据」插图 + 文字说明 |

### 5.2 列定义（从左到右）

| 列名 | 内容说明 | 宽度 | 固定 |
| :--- | :--- | :--- | :--- |
| 勾选列 | 复选框 | 48px | 左固定 |
| 采购单号 | `PO{日期}-{序号}`，可点击进入详情页，链接色展示 | 180px | 左固定 |
| 订单状态 | 状态 Tag（见§4.3配色） | 100px | — |
| 供应商 | `{Code} {Name}` | 200px | — |
| 入库仓库 | `{Code} {Name}` | 160px | — |
| 下单日期 | `YYYY-MM-DD` | 120px | — |
| 预计到货日期 | `YYYY-MM-DD`；无值展示 `-` | 130px | — |
| 商品种数 | `N 种`（如 `3 种`） | 80px | — |
| 采购总金额 | `¥1,234.56`（千分位，2位小数） | 130px | — |
| 最后修改时间 | `YYYY-MM-DD HH:mm:ss` | 170px | — |
| 操作 | 行内文字操作（见§4.4） | 120px | 右固定 |

### 5.3 状态 Tag 配色

| 状态 | 颜色语义 |
| :--- | :--- |
| 草稿 | 灰色 |
| 待审核 | 橙色 |
| 待入库 | 蓝色 |
| 部分入库 | 浅蓝色 |
| 已完成 | 绿色 |
| 已作废 | 红色 |

### 5.4 行内操作

行内操作使用**文字动作**，按状态动态展示（不满足条件的操作不渲染）：

| 状态 | 展示的操作 |
| :--- | :--- |
| 草稿 | **查看** / **编辑** / **提交审核** / **删除** |
| 待审核 | **查看** / **审核** / **驳回** / **作废** |
| 待入库 | **查看** / **创建入库单** / **作废** |
| 部分入库 | **查看** / **创建入库单** / **关闭订单** |
| 已完成 | **查看** |
| 已作废 | **查看** |

> **禁止渲染 disabled 操作**：操作不可用时直接不展示此入口，不展示禁用灰色文字。

## 六、二次确认弹窗规格

| 触发操作 | 弹窗标题 | 弹窗内容 | 确认按钮文案 | 取消按钮文案 |
| :--- | :--- | :--- | :--- | :--- |
| 删除（草稿） | 确认删除 | 删除后不可恢复，该采购订单将从系统中永久移除，确认删除？ | **确认删除** | 取消 |
| 提交审核 | 提交审核 | 提交后单据将进入待审核状态，确认提交？ | **确认提交** | 取消 |
| 审核通过 | 确认审核 | 审核通过后关键字段将锁定，不可再修改，确认审核通过？ | **确认审核** | 取消 |
| 驳回 | 确认驳回 | 驳回后单据将退回草稿，采购员可重新修改，确认驳回？ | **确认驳回** | 取消 |
| 作废 | 确认作废 | 作废后不可恢复，单据将进入已作废状态，确认作废？ | **确认作废** | 取消 |
| 关闭订单 | 确认关闭 | 关闭后剩余未入库数量将不再接收，确认关闭订单？ | **确认关闭** | 取消 |

## 七、Mock 数据规格

### 7.1 状态覆盖要求

| 状态 | 最少条数 | 备注 |
| :--- | :--- | :--- |
| 草稿 | 2 | — |
| 待审核 | 2 | — |
| 待入库 | 2 | — |
| 部分入库 | 2 | — |
| 已完成 | 2 | — |
| 已作废 | 1 | — |
| **总计** | **≥ 12** | — |

### 7.2 数据要求

- 供应商展示格式：`{Code} {Name}`，如 `VEND001 常青供应链科技有限公司`
- 入库仓库展示格式：`{Code} {Name}`，如 `WH001 广州中央仓`
- 采购单号格式：`PO20260419-0001`，序号连续递增
- 采购总金额范围：1,000 ~ 500,000 元，各状态数值有差异显示
- 所有日期字段使用虚构演示数据，不使用真实公司信息
