# 采购订单_Demo_新增编辑页

> **文档定位**：AI 生成采购订单新增/编辑页前端原型的输入文档，聚焦页面结构、控件类型、交互行为。
> **字段定义**：字段枚举值、计算规则、必填性等见《采购订单字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.1 | 2026-04-20

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 采购订单新增 / 编辑 |
| 页面目标 | 填写采购意图（向谁买、买什么、买多少、买多少钱） |
| 新增页路径 | `/purchase/orders/new` |
| 编辑页路径 | `/purchase/orders/{id}/edit` |
| 骨架结构 | 页头（标题）→ 表单区（头部信息卡片 + 商品明细卡片）→ 固定底部操作栏（返回列表 / 保存 / 保存并提交） |

---

## 二、页头

| 元素 | 规格 |
| :--- | :--- |
| 标题（新增模式） | `新增采购订单` |
| 标题（编辑模式） | `编辑采购订单 {采购单号}` |

页头不放业务操作按钮，操作按钮全部在底部操作栏。

## 三、固定底部操作栏

底部操作栏固定在页面右下角，不随页面滚动消失。

| 元素 | 规格 |
| :--- | :--- |
| 位置 | `fixed bottom-0 right-0`，内边距 `px-6 py-3` |
| 按钮顺序（从左到右） | `返回列表`（次操作）　`保存`（次操作）　`保存并提交`（主操作） |
| 按钮间距 | `gap-actions`（12px） |
| `保存` | 次要按钮样式（轻度边框） |
| `保存并提交` | 主要按钮样式（实心背景色） |
| `返回列表` | 文字按钮，无边框 |

## 四、头部信息卡片

卡片标题：**基本信息**

### 3.1 字段与控件

布局：4 列栅格（桌面端），自上而下排列。

| 字段 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| 供应商 | `SelectWithSearch` | 单选；展示格式 `{Code} {Name}`；数据来源：启用中的供应商档案；可清空；placeholder：`请选择供应商` |
| 入库仓库 | `SelectWithSearch` | 单选；展示格式 `{Code} {Name}`；数据来源：启用中的仓库档案；可清空；placeholder：`请选择入库仓库` |
| 下单日期 | `DatePicker` | 精确到日；默认值：今天；允许历史日期；**不允许未来日期**；不可清空 |
| 预计到货日期 | `DatePicker` | 精确到日；允许历史日期；允许未来日期；可清空；placeholder：`请选择预计到货日期` |
| 采购备注 | `Textarea` | 最多 200 个字符；默认展示 3 行，支持自动变高；允许换行；占满当前行剩余宽度（`md:col-span-2 xl:col-span-4`）；placeholder：`请输入采购备注（选填）` |

### 3.2 字段状态（编辑模式）

> 编辑页区分草稿态和审核后状态，字段可编辑性不同：

| 字段 | 草稿态（编辑页） | 审核通过后（不可编辑） |
| :--- | :--- | :--- |
| 供应商 | `SelectWithSearch`（可编辑） | `ReadonlyText`（只读，展示 `{Code} {Name}`） |
| 入库仓库 | `SelectWithSearch`（可编辑） | `ReadonlyText`（只读） |
| 下单日期 | `DatePicker`（可编辑） | `ReadonlyText`（只读） |
| 预计到货日期 | `DatePicker`（可编辑） | `DatePicker`（仍可编辑，审核后不锁定） |
| 采购备注 | `Textarea`（可编辑） | `Textarea`（仍可编辑，审核后不锁定） |

> **注意**：审核通过后理论上进入「待入库」或「部分入库」不走编辑页，此处供系统兜底控制参考。

## 五、商品明细卡片

卡片标题：**商品明细**

### 4.1 表格操作

| 操作 | 位置 | 说明 |
| :--- | :--- | :--- |
| **选择商品** | 工具条左侧 | 打开商品选择弹窗（支持搜索+多选）；确认后批量写入明细行 |
| **删除行** | 行内末尾操作列 | 删除对应商品行，无需二次确认（草稿态允许反复增删） |

- 至少保留 1 行不可删除；删除最后一行时提示「至少保留一行商品明细」
- 最多 50 行，超出时「选择商品」按钮 disabled 并提示「商品明细最多 50 行」

### 4.2 明细行列定义

| 列名 | 控件类型 | 详细规格 |
| :--- | :--- | :--- |
| 序号 | `ReadonlyText` | 自动生成，从 1 起 |
| 商品编码 | `ReadonlyText` | 由商品选择弹窗写入，行内不可直接编辑；如需更换商品，删除本行后重新选品 |
| 商品名称 | `ReadonlyText` | 选中商品编码后自动带出，快照只读展示；未选时展示 `-` |
| 商品条码 | `ReadonlyText` | 选中商品编码后自动带出，快照只读展示；商品无条码时展示 `-` |
| 规格型号 | `ReadonlyText` | 选中商品后自动带出，空值展示 `-` |
| 单位 | `ReadonlyText` | 选中商品后自动带出，空值展示 `-` |
| 采购数量 | `NumberInput` | 正整数（仅整数）；最小值 1；最大值 9,999,999；不允许负数；不展示步进按钮 |
| 单价（含税） | `NumberInput` | 最多保留 2 位小数；最小值 0；不允许负数；前缀 `¥`；选中商品后从商品档案带出默认采购价（允许人工修改） |
| 税率 | `Select` | 枚举单选，见字段清单；默认空；可清空；不支持搜索 |
| 金额（含税） | `ReadonlyText` | `采购数量 × 单价`，实时计算，2 位小数，千分位展示 |
| 行备注 | `Input` | 最多 100 个字符；任意字符；placeholder：`行备注（选填）`；可清空 |
| 操作 | 文字按钮 | 仅展示「删除」 |

### 4.3 商品选择弹窗

> 本模块使用**通用商品选择弹窗组件**，完整规格见：
> [`全局约束/通用组件-商品选择弹窗.md`](../全局约束/通用组件-商品选择弹窗.md)

**采购订单调用配置**：

| 参数 | 本模块取值 |
| :--- | :--- |
| `mode` | `multi`（多选） |
| `maxSelectCount` | `50 - 当前明细行数`（动态计算剩余可选数） |
| `showPriceColumn` | `true` |
| `priceColumnLabel` | `默认采购价` |
| `defaultPriceField` | `defaultPurchasePrice` |

**确认后写入特殊规则**：
- 🔒 商品编码、商品名称、商品条码、规格型号、单位：写入后行内只读，不可修改；如需更换商品，删除该行后重新选品
- ✏️ 单价（含税）：预填默认采购价，**允许人工修改**
- ✏️ 采购数量：默认写入 `1`，**允许人工修改**
- ⬛ 税率：默认空，用户在行内手动选择
- 弹窗关闭后，焦点自动移至**新添加第一行**的「采购数量」字段

### 4.4 汇总行（固定于表格底部）

| 汇总项 | 展示方式 |
| :--- | :--- |
| 商品种数 | `共 N 种商品` |
| 采购总数量 | `合计：N 件` |
| 采购总金额 | `¥ N（千分位，2 位小数）` |

> 汇总行样式：背景 `bg-zinc-100`，字体加粗，固定在明细列表底部。

## 六、校验规则

### 6.1 头部字段校验

| 校验对象 | 校验条件 | 触发时机 | 错误提示 | 是否阻断提交 |
| :--- | :--- | :--- | :--- | :--- |
| 供应商 | 必选，不可为空 | 点击「提交审核」时全量校验 | 字段下方红字：`请选择供应商` | 是 |
| 入库仓库 | 必选，不可为空 | 点击「提交审核」时全量校验 | 字段下方红字：`请选择入库仓库` | 是 |
| 下单日期 | 必填；不允许未来日期 | 值变更后失焦 + 提交时全量校验 | 字段下方红字：`下单日期不能晚于今天` | 是 |

### 6.2 商品明细校验

| 校验对象 | 校验条件 | 触发时机 | 错误提示 | 是否阻断提交 |
| :--- | :--- | :--- | :--- | :--- |
| 商品明细行 | 至少 1 行；最多 50 行 | 点击「提交审核」时 | Toast：`商品明细不可为空，请至少添加一行商品` | 是 |
| 商品编码（每行） | 必选，不可为空 | 値变更后失焦 | 行内红字：`请选择商品` | 是 |
| 采购数量（每行） | 必填，且 > 0 | 值变更后失焦 | 行内红字：`采购数量不可为 0` | 是 |
| 单价（每行） | 必填，且 ≥ 0 | 值变更后失焦 | 行内红字：`请填写单价` | 是 |

## 七、操作行为说明

| 操作 | 行为 |
| :--- | :--- |
| 保存草稿 | 校验必填字段（宽松校验：仅阻断明显错误）；保存成功后留在当前页，Toast 提示「保存成功」 |
| 提交审核 | 全量校验（严格校验）；校验通过后跳转至该订单详情页；Toast 提示「已提交审核，等待审核确认」 |
| 返回列表 | 若有未保存改动，弹 Modal 确认「当前有未保存的内容，确认离开？」；确认后跳转列表页 |
| 选择商品 | 打开商品选择弹窗；用户搜索+多选确认后，每件商品对应一条明细行，内容全部写入（采购数量默认 1，单价来自商品档案） |

## 八、模式差异汇总

| 差异项 | 新增模式 | 编辑模式（草稿） |
| :--- | :--- | :--- |
| 页面标题 | `新增采购订单` | `编辑采购订单 {采购单号}` |
| 采购单号 | 不展示（保存后生成） | 在页头副标题展示 |
| 表单数据 | 全部为空（下单日期除外） | 回显已保存数据 |
| 提交后跳转 | 跳转到新单据详情页 | 跳转到该单据详情页 |
| 适用状态 | — （仅全新单据） | 仅「草稿」状态可进入编辑页 |

## 九、反馈文案

| 操作 | 反馈类型 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 保存并提交成功 | Toast | 已提交审核，等待审核确认 |
| 离开未保存 | Modal | 当前有未保存的内容，确认离开？（确认离开 / 继续编辑） |
| 商品行超限 | Toast | 商品明细最多 50 行 |
| 至少保留1行 | Toast | 至少保留一行商品明细 |
