## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 执行采购退货出库 |
| 页面目标 | 仓管员在此页面录入退货商品的实际出库数量，点击确认后正式扣减库存、冲减应付并生成 FL |
| 新增路径 | `/purchase/return-shipments/new?source_id={pr_id}` (PR下推跳转) |

| 骨架结构 | 页头 → 表单区（基本信息卡片 + 商品明细卡片） → 固定底部操作栏 |

## 二、固定底部操作栏

| 元素 | 规格 |
| :--- | :--- |
| 位置 | `fixed bottom-0 right-0` |
| 按钮顺序 | `返回列表`（次要操作）　`保存草稿`（次要操作）　`确认出库`（主要操作） |
| 返回列表 | 点击后若表单已被修改，弹出二次确认「数据尚未保存，确定返回列表？」 |

## 四、商品明细卡片

卡片标题：**出库明细清单**。

### 4.1 明细工具条

- 无“选择商品”按钮：PRO 明细行必须完全来源于 PR 明细，禁止手动添加其他商品。
- 行“删除”按钮：支持行内删除。当某商品本次不出库时，仓管员可将该行从本次 PRO 单据中删除，仅代表本次出库不含该商品，PR 剩余可出库数量不受影响。

### 4.2 表格列定义（从左到右）

| 列名 | 呈现规格与控件类型 | 详细交互说明 |
| :--- | :--- | :--- |
| 序号 | `ReadonlyText` | 自动生成，从 1 起递增 |
| 商品编码 | `ReadonlyText` | 继承只读 |
| 商品名称 | `ReadonlyText` | 快照只读 |
| 商品条码 / 规格 | `ReadonlyText` | 快照只读，合并单列展示 |
| 单位 | `ReadonlyText` | 快照只读 |
| PR退货数量 | `ReadonlyText` | 原 PR 已确认退货数量，仅做信息参考 |
| 已出库数量 | `ReadonlyText` | 同一 PR 行已确认 PRO 出库数量 |
| 可出库数量 | `ReadonlyText` | 本次出库数量的核心上限校验基准，高亮展示 |
| 出库数量 | `NumberInput` | 必填，正整数。默认回填为可出库数量，用户可改小 |
| 单价（含税） | `ReadonlyText` | 继承 PR 的含税单价，格式：`¥{价格}` |
| 金额（含税） | `ReadonlyText` | 系统实时计算并展示：`出库数量 × 单价（含税）` |
| 行备注 | `Input` | 字符上限 100 字。提示：`行备注（选填）` |
| 操作 | 文字按钮 | 删除。点击后从本次 PRO 中移除该商品行 |

### 4.3 列表底部汇总信息

明细表格右下方展示统计汇总，为只读计算：
- 商品种数：`明细行数`
- 出库总数量：`所有行“出库数量”之和`
- 出库总金额：`所有行“金额（含税）”之和`，展示格式如 `¥26,500.00`

## 五、页面字段联动与控制规则

### 5.1 智能回填联动

1. **出库数量默认回填**：从 PR 下推创建 PRO 时，系统默认将每行 `出库数量` 回填为当前 `可出库数量`。
2. **金额实时重算**：一旦明细行 `出库数量` 发生变化，系统实时重新计算本行金额，并更新底部出库总金额和出库总数量。

### 5.2 强控拦截交互控制

1. **超出 PR 剩余量警告**：当用户填入的 `出库数量` 大于 `可出库数量` 时，输入框下方展示红色提示「不能超过可出库数量({可出库数量})」，并置灰“确认出库”按钮。
2. **现存不足警告**：当 `出库数量` 大于当前仓库该商品现存时，输入框下方展示红色提示「当前仓库现存不足」，并置灰“确认出库”按钮。
3. **明细为空校验**：若仓管员删除全部商品行，明细卡片展示「暂无出库商品，请重新从采购退货单下推」，并置灰“确认出库”和“保存草稿”按钮。
