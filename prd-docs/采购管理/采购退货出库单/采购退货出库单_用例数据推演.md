## 3. 推演案例

### 案例 1：正常退货出库流程（PR与PRO 1:1）

> **目标**：验证 PRO 确认出库后，库存、应付、FL 和 PR 状态全部正确更新。

**推演步骤**：

| 步骤 | 操作人/系统 | 操作 | 数据变化（字段 = 旧值 → 新值） | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 仓管员 | 从 PR20260707-0001 下推创建 PRO | `PRO单号 = PRO20260707-0001（新生成）`<br>`单据状态：（无）→ 草稿`<br>`出库数量：继承 PR 只读 = SKU001: 3, SKU002: 5`<br>`金额（含税）：系统计算 = SKU001: 150.00, SKU002: 60.00` | 单号格式校验合格；来源继承无遗漏；出库数量锁定只读不可改 |
| 2 | 仓管员 | 确认并录入出库日期与备注 | `出库日期 = 2026-07-07`<br>`出库备注 = 采购退货出库实物交接` | 录入基本出库信息 |
| 3 | 仓管员 | 点击“确认出库” | `出库状态：草稿 → 已确认`<br>`确认人 = 仓管员_张三`<br>`确认时间 = 2026-07-07 16:10:15` | 校验通过：现存足够（3≤25，5≤40）且 1:1 数量一致 |
| 4 | 系统自动 | 扣减仓库商品现存并生成 FL | `WH001 SKU001现存：25 → 22`<br>`WH001 SKU001可用：25 → 22`<br>`WH001 SKU002现存：40 → 35`<br>`WH001 SKU002可用：40 → 35`<br>`库存流水号1 = FL20260707-00000001（数量 -3）`<br>`库存流水号2 = FL20260707-00000002（数量 -5）` | 库存扣减方向正确；流水前缀与流向符号正确 |
| 5 | 系统自动 | 冲减供应商应付 | `S001应付余额：740.00 → 530.00`<br>`应付冲减金额 = 210.00` | 冲减金额与 PRO 出库总金额（150+60）一致 |
| 6 | 系统自动 | 回写 PR 状态 | `PR 关联出库状态：未出库 → 已出库` | PR 标记已出库，不再允许重复生成 PRO |

## 6. 异常与边界案例

### 案例 2：现存不足阻断

| 场景 | 操作 | 预期结果 |
| :--- | :--- | :--- |
| 假设 WH001 SKU001 现存变更为 2，而 PRO 继承出库量为 3 | 点击确认出库 | 阻断，提示「商品 华强北特种接插件 确认失败：当前仓库现存不足，无法退货出库」 |
| 阻断后 | 查看 PRO 状态和数据 | 仍为草稿，不扣库存、不冲应付、不生成 FL |

### 案例 3：草稿删除/作废

| 操作 | 状态变化 | 数据影响 |
| :--- | :--- | :--- |
| 删除草稿 PRO | 草稿 → 物理消失 | 关联的 PR 恢复为“未出库”状态，可重新下推 PRO；不影响库存与应付 |
| 作废草稿 PRO | 草稿 → 已作废 | 保留 PRO 只读记录；关联的 PR 恢复为“未出库”状态，可重新下推 PRO；不影响库存与应付 |

### 案例 4：边界与并发

| 场景 | 操作 | 预期系统行为 |
| :--- | :--- | :--- |
| 来源 PR 为草稿或已作废 | 下推 PRO | 阻断，提示仅已确认 PR 可下推 |
| 已生成过 PRO 的 PR 再次尝试下推 | 重复下推 | 阻断，提示「下推失败，该采购退货单已生成过采购退货出库单」 |
| 已确认 PRO 尝试编辑 | 访问页面 | 阻断并跳转详情页，提示当前状态不可编辑 |
| 草稿 PRO 保存 | 保存草稿 | 不扣现存、不冲应付、不生成 FL |
