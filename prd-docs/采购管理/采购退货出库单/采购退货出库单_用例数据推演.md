## 3. 推演案例

### 案例 1：分批退货出库（PR退8件，本次PRO出6件）

> **目标**：验证 PRO 确认出库后，库存、应付、FL 和 PR 出库进度全部正确。

**推演步骤**：

| 步骤 | 操作人/系统 | 操作 | 数据变化（字段 = 旧值 → 新值） | 验证点 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | 仓管员 | 从 PR20260707-0001 下推创建 PRO | `PRO单号 = PRO20260707-0001（新生成）`<br>`单据状态：（无）→ 草稿`<br>`商品编码：继承 = SKU001`<br>`可出库数量：继承 = 8`<br>`出库数量：默认 = 8` | 单号格式校验合格；来源继承无遗漏 |
| 2 | 仓管员 | 录入本次实际出库数量 | `出库数量：8 → 6`<br>`金额（含税）：400.00 → 300.00` | 行金额公式重算：`6 × 50.00 = 300.00` |
| 3 | 仓管员 | 点击“确认出库” | `出库状态：草稿 → 已确认`<br>`确认人 = 仓管员_张三（新记录）`<br>`确认时间 = 2026-07-07 16:10:15（新记录）` | 校验通过：6 ≤ 可出库 8，且 6 ≤ 现存 21 |
| 4 | 系统自动 | 扣减仓库商品现存并生成 FL | `WH001 SKU001现存：21 → 15`<br>`WH001 SKU001可用：21 → 15`<br>`库存流水号 = FL20260707-00000001（新生成）`<br>`流水类型 = 采购退货出库`<br>`流水数量 = -6` | 库存扣减方向正确；FL 前缀符合 context/05 |
| 5 | 系统自动 | 冲减供应商应付 | `S001应付余额：400.00 → 100.00`<br>`应付冲减金额 = 300.00` | 冲减金额与 PRO 出库总金额一致 |
| 6 | 系统自动 | 回写 PR 出库进度 | `PR SKU001已出库数量：0 → 6`<br>`PR SKU001可出库数量：8 → 2` | 累计出库数量是累加而非覆盖 |

## 6. 异常与边界案例

### 案例 2：超出 PR 可出库数量阻断


| 场景 | 操作 | 预期结果 |
| :--- | :--- | :--- |
| PR 剩余可出库数量为 2 | 草稿 PRO 输入出库数量 3 并确认 | 阻断，提示「出库数量超过采购退货单剩余可出库数量 (2件)」 |
| 阻断后 | 查看库存和应付 | 现存、可用、应付余额、FL 均不变化 |

### 案例 3：现存不足阻断


| 场景 | 操作 | 预期结果 |
| :--- | :--- | :--- |
| WH001 SKU001 现存仅 1，PRO 出库数量为 2 | 点击确认出库 | 阻断，提示「当前仓库现存不足，无法退货出库」 |
| 阻断后 | 查看 PRO 状态 | 仍为草稿，不生成 FL，不冲减应付 |

### 案例 4：草稿删除/作废


| 操作 | 状态变化 | 数据影响 |
| :--- | :--- | :--- |
| 删除草稿 PRO | 草稿 → 物理消失 | 不回写 PR；不影响库存；不影响应付 |
| 作废草稿 PRO | 草稿 → 已作废 | 保留记录；不回写 PR；不影响库存；不影响应付 |

### 案例 5：边界与并发


| 场景 | 操作 | 预期系统行为 |
| :--- | :--- | :--- |
| 来源 PR 为草稿或已作废 | 下推 PRO | 阻断，提示仅已确认 PR 可下推 |
| 两人同时确认同一 PR 行的 PRO | 后确认者提交 | 系统重新计算可出库数量，超出则阻断 |
| 已确认 PRO 尝试编辑 | 访问编辑页 | 阻断并跳转详情页，提示当前状态不可编辑 |
| 草稿 PRO 保存 | 保存草稿 | 不扣现存、不冲应付、不生成 FL |
