# 库存流水查询_业务规则规格

> **版本**：V1.0 | 2026-07-04
> **前置文档**：《库存流水查询主PRD》《库存流水查询字段清单》

---

## 一、查询与筛选逻辑

本页面支持多条件组合筛选，系统后台在执行 SQL 查询时应遵循以下多字段组合筛选及匹配逻辑：

### 1.1 筛选字段组合逻辑

| 筛选字段 | 匹配方式 | 空值处理 | 校验与阻断规则 |
| :--- | :--- | :--- | :--- |
| **仓库** | 精确匹配 (`=`) | 默认不限，展示该用户管辖下的全部仓库流水 | 必须通过当前登录用户的仓库权限过滤（见§3.2） |
| **商品** | 精确匹配 (`=`) | 默认不限，展示所有商品的流水 | 支持按编码或名称进行下拉联想搜索 |
| **单据类型** | 精确匹配 (`=`) | 默认不限，展示所有变动类型的流水 | 界面下拉单选，包含全部类型 |
| **变动方向** | 符号逻辑 (`> 0` 或 `< 0`) | 默认不限，不进行数量正负值过滤 | 选入库过滤数量 > 0，选出库过滤数量 < 0 |
| **来源单号** | 右模糊匹配 (`LIKE 'PO%'` 等) | 默认不限，不进行单号过滤 | 字符输入，不限制前缀大小写（系统自动转为大写匹配） |
| **日期范围** | 闭区间范围匹配 (`BETWEEN AND`) | 默认不限，展示历史所有流水记录 | **强控校验**：选择的开始日期到结束日期，天数跨度不能超过 **365 天**。超出时页面进行红色报错，阻断查询按钮，以防止大范围扫表造成性能瓶颈。 |

### 1.2 组合筛选默认行为
- 进入页面时，默认不自动进行大范围历史查询（或默认只自动查询**近 30 天**的流水记录），页码初始化为第 1 页。
- 点击“重置”按钮时，清空所有输入框及下拉选择的值，日期范围恢复默认。

## 二、展示规则与映射规格

为保证库存日志的可读性，系统在前端展现流水记录时必须满足以下转换：

### 2.1 变动类型与来源单号前缀映射

系统根据底层的 `来源单号` 前缀或单据类型，自动在前端列表中渲染“变动类型”文案：

| 来源单号前缀 | 底层单据类型 | 前端渲染变动类型文案 | 流水变动方向 (默认符号) |
| :--- | :--- | :--- | :---: |
| `PI` | 采购入库单 | **采购入库** | 增加 (+) |
| `PRO` | 采购退货出库单 | **采购退货出库** | 减少 (-) |
| `SOO` | 销售出库单 | **销售出库** | 减少 (-) |
| `SR` | 销售退货入库单 | **销售退货入库** | 增加 (+) |
| `RS` | 零售结账单 | **零售单成交** | 减少 (-) |
| `TR` (出库) | 调拨出库确认 | **调拨出库** | 减少 (-) |
| `TR` (入库) | 调拨入库确认 | **调拨入库** | 增加 (+) |
| `BL` | 报损单 | **报损出库** | 减少 (-) |
| `CK` (盈) | 盘点单盘盈 | **盘盈入库** | 增加 (+) |
| `CK` (亏) | 盘点单盘亏 | **盘亏出库** | 减少 (-) |

### 2.2 变动数量的色彩管理
为了帮助用户快速捕捉库存吞吐情况，列表中的 `变动数量` 需加带正负号，并应用以下色彩标准：
- **正数（入库）**：格式如 `+10`，字体颜色显示为 **绿色** (Green)
- **负数（出库）**：格式如 `-5`，字体颜色显示为 **黑色** (Black) 或 **深灰色**，不使用红色（红色预留给强阻断报错）

## 三、权限与数据范围隔离规则

### 3.1 角色操作权限

| 角色 | 组合检索 | 来源单号跳转 | 导出 Excel | 新增/编辑/删除/作废 |
| :--- | :---: | :---: | :---: | :---: |
| **仓管员** | ✅ | ✅ | ✅ | ❌ |
| **采购员** | ✅ | ✅ | ✅ | ❌ |
| **财务员** | ✅ | ✅ | ✅ | ❌ |
| **系统管理员** | ✅ | ✅ | ✅ | ❌ |

### 3.2 仓库级数据可见性控制 (行级权限隔离)

*   **规则 P_DATA_FL01 (仓管隔离)**：非管理员/财务角色的仓管员在进入“库存流水查询”页面时，系统必须自动在其查询条件和结果集中注入该仓管员管辖的“仓库编码范围”。
    *   *控制机制*：列表只能出现其管辖库房的流水，仓库筛选下拉框中仅能选择其拥有管辖权的仓库，不允许其查询其他仓库的数据。
*   **规则 P_DATA_FL02 (采购员范围限制)**：采购员在进入该页面时，系统仅展示“变动类型”为“采购入库”或“采购退货出库”的流水记录，隐藏销售、零售、调拨等无关流水，防范商业敏感数据泄露。
*   **规则 P_DATA_FL03 (全局可见性)**：财务和管理员在查询流水时，全局所有仓库和所有单据类型的流水数据全量可见，无隔离控制。
