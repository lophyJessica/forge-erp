# 库存流水查询_Demo_列表页

> **文档定位**：AI 生成库存流水查询列表页前端原型的输入文档，聚焦页面结构、过滤控件、表格列和点击跳转行为。
> **字段定义**：字段取值及筛选规则详见《库存流水查询字段清单》，本文件只写页面呈现规格，严禁重复定义。
> **版本**：V1.0 | 2026-07-04

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 库存流水查询 |
| 页面目标 | 提供对全系统所有商品历史库存变动明细的组合查询与导出功能，实现账实可追溯 |
| 页面路径 | `/inventory/flows` |
| 骨架结构 | 查询卡片 + 列表卡片（两卡片结构，无状态 Tab 栏） |

---

## 二、状态 Tab 栏

> ⚠️ **不适用**：本模块为只读审计流水台账，商品流水的变动数据无“草稿/待审核/已审核”等业务生命周期状态，不涉及单据状态过滤。因此页面不提供状态 Tab 栏，直接展示查询卡片。

---

## 三、查询区

### 3.1 布局说明
- 桌面端 6 列栅格，查询条件默认展示 1 行（前 6 个条件），展开后展示第 2 行。右侧固定操作组：`展开/收起` $\rightarrow$ `重置` $\rightarrow$ `搜索`。

### 3.2 查询条件

| 字段 | 控件类型 | 规格说明 |
| :--- | :--- | :--- |
| 仓库 | `SelectWithSearch` | 联想下拉单选。支持按仓库编码或名称搜索，展示格式：`{Code} {Name}`。仓管员角色默认被系统限制可选范围（见《规则规格》隔离控制） |
| 商品 | `SelectWithSearch` | 联想下拉单选。支持按商品编码或名称搜索，展示格式：`{Code} {Name}` |
| 单据类型 | `Select` | 枚举下拉单选，含「全部」选项。不支持搜索 |
| 变动方向 | `Select` | 下拉单选。选项包括：`全部` / `入库（+）` / `出库（-）` |
| 日期范围 | `DateRange` | 默认范围为空。**强控规则**：选择天数跨度不得超过 365 天。若超出，输入框下方标红并报错提示「查询时间跨度不可超过 365 天」 |
| 来源单号 | `Input` | 普通文本录入，支持按前缀或单号模糊匹配搜索 |

---

## 四、工具条

工具条位于查询卡片与表格之间，左侧仅放全局只读操作，右侧放表格操作。由于本页面为只读台账，**工具条左侧不提供「新增」等编辑按钮**。

| 按钮 | 位置 | 状态 | 说明 |
| :--- | :--- | :--- | :--- |
| **导出** | 右侧 | 动态 | 导出当前筛选过滤出的库存流水 Excel 明细。当列表无数据（展示空状态）时，本按钮为 **Disabled (置灰)** 状态不可点击 |

---

## 五、表格

### 5.1 表格行为

| 行为 | 规格 |
| :--- | :--- |
| 行勾选 | 不支持（流水仅供查看与导出，不支持批量编辑或批量确认） |
| 列宽拖拽 | 支持 |
| 固定列 | 库存流水号左固定；来源单号及发生时间右固定 |
| 分页 | 默认每页 20 条，支持每页条数切换（20/50/100） |
| 空状态 | 展示「暂无数据」插图，并在下方展示「在当前筛选条件下，没有找到对应的库存流水记录」 |

### 5.2 列定义（从左到右）

| 列名 | 内容说明 | 宽度 | 固定 |
| :--- | :--- | :--- | :--- |
| 库存流水号 | `FL{日期}-{序号}`，只读文本展示 | 180px | 左固定 |
| 发生时间 | 写入流水的准确系统时间，格式：`YYYY-MM-DD HH:mm:ss` | 170px | — |
| 仓库 | 发生仓库，展示为 `{Code} {Name}` | 160px | — |
| 商品编码 | 商品编码，只读 | 120px | — |
| 商品名称 | 商品名称快照，只读 | 180px | — |
| 规格型号 | 商品规格快照，无值显示 `-` | 130px | — |
| 单位 | 商品计量单位快照 | 80px | — |
| 变动类型 | 系统归类映射的业务文案（如采购入库、销售出库等） | 120px | — |
| **变动数量** | 带正负符号整数。正数带 `+` 且显示为 **绿色**；负数带 `-` 且显示为 **黑色** | 110px | — |
| 变动后现存量 | 变动发生后的仓库最新现存，大号字展示 | 120px | — |
| **来源单号** | 触发流水的单据号。**链接高亮显示**，双击/点击可在新窗口打开对应执行层单据详情页 | 180px | 右固定 |
| 操作人 | 提交/确认该单据的操作用户姓名 | 110px | 右固定 |

## 六、二次确认弹窗规格

> ⚠️ **不适用**：本页面仅供组合查询与导出，无任何删除、确认、作废或修改动作，因此本页面不涉及任何二次确认弹窗。

---

## 七、Mock 数据规格

### 7.1 数据条数要求
页面进入时，应提供至少 **10 条** Mock 数据以展示不同单据类型引起流水变化时的排版效果。

### 7.2 变动类型数据要求
Mock 数据的 `变动类型` 列必须覆盖以下几种典型情况：
- 包含至少 2 条 `采购入库` 记录（单号 PI...，变动数量为 `+`）
- 包含至少 2 条 `销售出库` 记录（单号 SOO...，变动数量为 `-`）
- 包含至少 2 条 `零售单成交` 记录（单号 RS...，变动数量为 `-`）
- 包含至少 1 条 `调拨出库` / `调拨入库` 记录（单号 TR...）
- 包含至少 1 条 `报损出库` 记录（单号 BL...）

| 操作 | 反馈类型 | 文案 |
| :--- | :--- | :--- |
| 查询失败 | Toast | 查询失败，{具体失败原因} |
| 导出成功 | Toast | 流水明细导出成功，共导出 N 条数据 |
