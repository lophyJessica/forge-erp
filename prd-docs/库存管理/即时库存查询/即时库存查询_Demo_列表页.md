# 即时库存查询_Demo_列表页

> **文档定位**：AI 生成即时库存查询列表页前端原型的输入文档，聚焦页面结构、过滤控件、表格列（现存/占用/可用共屏）和预警样式。
> **字段定义**：字段取值及计算公式见《即时库存查询字段清单》，本文件只写页面呈现规则，严禁重复定义。
> **版本**：V1.0 | 2026-07-04

---

## 一、页面概述

| 项目 | 内容 |
| :--- | :--- |
| 页面名称 | 即时库存查询 |
| 页面目标 | 提供对全系统所有商品各仓库、各批次实时现存、占用、可用库存数量的组合查询与导出功能 |
| 页面路径 | `/inventory/instant-stocks` |
| 骨架结构 | 查询卡片 + 列表卡片（两卡片结构，无状态 Tab 栏） |

---

## 二、状态 Tab 栏

> ⚠️ **不适用**：即时库存代表实时的数量状态视图，没有业务状态（如草稿/已确认）过滤需求。因此页面不提供状态 Tab 栏，直接展现查询卡片。

---

## 三、查询区

### 3.1 布局说明
- 桌面端 6 列栅格，默认展示 1 行（4 个条件），Switch 开关单列宽度。右侧操作组：`重置` $\rightarrow$ `搜索`。

### 3.2 查询条件

| 字段 | 控件类型 | 规格说明 |
| :--- | :--- | :--- |
| 仓库 | `SelectWithSearch` | 联想下拉单选。从启用中仓库选择，展示格式：`{Code} {Name}`。仓管员角色默认被系统限制可选范围（见《规则规格》隔离控制） |
| 商品 | `SelectWithSearch` | 联想下拉单选。从启用中商品选择，展示格式：`{Code} {Name}` |
| 批次号 | `Input` | 文本输入，支持右模糊匹配搜索。提示：`请输入批次号` |
| 不显示零库存 | `Switch` | 开关控件，默认值为 `关闭` (即显示所有建立过库存关系的商品)。勾选开启后，系统自动过滤隐藏现存量 $\le$ 0 的记录行 |

---

## 四、工具条

工具条位于查询卡片与表格之间，左侧无业务操作，右侧放表格级操作。本页面为只读台账，**工具条左侧不提供「新增」等编辑按钮**。

| 按钮 | 位置 | 状态 | 说明 |
| :--- | :--- | :--- | :--- |
| **导出** | 右侧 | 动态 | 导出当前筛选过滤出的即时库存明细 Excel。当列表无数据（展示空状态）时，本按钮为 **Disabled (置灰)** 状态不可点击 |

---

## 五、表格

### 5.1 表格行为

| 行为 | 规格 |
| :--- | :--- |
| 行勾选 | 不支持（只读查询，不支持批量操作） |
| 列宽拖拽 | 支持 |
| 固定列 | 商品编码、商品名称列左固定；可用量及预警 Tag 列右固定 |
| 分页 | 默认每页 20 条，支持每页条数切换（20/50/100） |
| 空状态 | 展示「暂无数据」插图，并在下方展示「在当前筛选条件下，没有找到对应的即时库存记录」 |

### 5.2 列定义（从左到右）

| 列名 | 内容说明 | 宽度 | 固定 |
| :--- | :--- | :--- | :--- |
| 商品编码 | 商品唯一标识编码 | 120px | 左固定 |
| 商品名称 | 商品档案中文名称 | 180px | 左固定 |
| 规格型号 | 规格快照，无值显示 `-` | 130px | — |
| 单位 | 计量单位快照 | 80px | — |
| 仓库 | 存放仓库，展示为 `{Code} {Name}` | 160px | — |
| 批次号 | 实物对应的生产或入库批次号，无批次商品显示 `-` | 140px | — |
| 现存量 | 物理实物数量。若现存 < 0 发生倒挂，数值呈 **红色** 展示 | 110px | — |
| 占用量 | 销售订单锁定数量。只读展示 | 110px | — |
| **可用量** | 可承诺销售数量。**只读大字展示**。当 `可用量` < `安全库存量` 时，数值展示为 **红色加粗** 样式 | 110px | 右固定 |
| 安全库存量 | 商品档案定义的安全存量参考值，无值展示 `-` | 120px | — |
| 最近变动时间 | 该仓库该商品最后一次发生库存流水变动的时间 | 170px | — |
| **预警标识** | 当 $可用量 < 安全库存量$ 时，本列渲染黄色高亮 Tag **`⚠️低于安全库存`**；否则为空不展示 | 130px | 右固定 |

## 七、Mock 数据规格

### 7.1 数据条数要求
页面进入时，应提供至少 **5 条** Mock 数据以展示不同商品在不同仓库、不同批次以及预警状态下的表格效果。

### 7.2 数据展现要求
- 商品 `SKU001`（华强北特种接插件）在 `WH001` 下有两条记录：批次 `20260701-A1` (现存 25，占用 5，可用 20)；批次 `20260704-A1` (现存 6，占用 0，可用 6)。
- 商品 `SKU002`（贴片电容）在 `WH001` 下（现存 50，占用 0，可用 50，安全库存 100），由于低于安全库存，可用量“50”渲染为红色加粗，预警标识列展示 `⚠️低于安全库存` 黄色 Tag。
- 商品 `SKU003`（二极管）在 `WH002` 下现存为 0。当开启“不显示零库存”开关后，该行被过滤剔除。
