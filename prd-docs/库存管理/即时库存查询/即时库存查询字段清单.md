> **文档定位**：即时库存查询页面所有字段的唯一定义来源。下游文档只引用字段名称，不重复定义字段含义及计算规则。

> **引用规则**：
> - 本模块为只读查询台账，字段清单中剔除“新增页”和“编辑页”列，改为「查询条件」、「列表展示」及「详情展示」
> - 下游文档（主PRD、Demo PRD）只引用本文件定义的字段名，严禁重复定义字段本体

## 一、查询条件字段

| 字段名称 | 控件类型 | 取值说明 | 必填性 | 默认值 | 过滤匹配规则 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 仓库 | `SelectWithSearch` | 来源于启用中的仓库档案；展示格式：`{Code} {Name}` | 选填 | 空（全部仓库） | 精确匹配 | 筛选特定仓库商品即时库存 |
| 商品 | `SelectWithSearch` | 来源于启用中的商品档案；支持编码或名称联想检索 | 选填 | 空（全部商品） | 精确匹配 | 筛选特定商品即时库存 |
| 批次号 | `Input` | 文本输入，由出入库单据执行时生成记账 | 选填 | 空 | 模糊匹配 | 筛选特定批次实物库存 |
| 不显示零库存 | `Switch` | 布尔型：`true=隐藏现存量<=0的行` / `false=展示全部` | 选填 | `false`（默认显示） | 数量过滤 | 勾选后，后台 SQL 自动拼接过滤 `current_qty > 0` |

| 字段名称 | 字段来源 | 取值说明 | 必填性 | 展示说明 | 字段说明 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 查询生成时间 | 系统生成 | 当前查询请求响应的系统时间：`YYYY-MM-DD HH:mm:ss` | 系统自动 | 列表底置只读 | 反映当前即时库存计算的新鲜度截止点 | - |

## 四展示与引用规则

- **预警高亮**：若 `可用量` < `安全库存量`，可用量数值以前端 **红色加粗** 字体显示，并在单元格最右侧显示黄色警告 Tag `⚠️低于安全库存`。
- **负库存警示**：若出现 `现存量` < 0（即发生库存倒挂），现存量数值以 **红色** 字体展示并发出视觉警示，但后台接口返回的可用数统一为 0。
- **批次合并展示**：启用了批次号管理的 SKU 按“商品 + 仓库 + 批次”分行独立展示；未启用的商品其批次列展示为 `-`。
