# 仓库档案_前端Demo版PRD

**version**：v1.0.0  
**date**：2026-04-18  
**description**：仓库档案前端页面规范，统一承接仓库列表页、查看页、新增页、编辑页。字段定义以《仓库档案字段清单》为准；本文档只定义页面结构、控件类型、显隐逻辑、交互与 Mock 数据要求。

---

## 一、页面清单

| 页面 | 目标 |
| :--- | :--- |
| 仓库列表页 | 查询仓库、查看状态、进入新增/编辑/详情 |
| 仓库详情页 | 查看仓库基础资料和说明信息 |
| 新增仓库页 | 创建新仓库档案 |
| 编辑仓库页 | 修改已有仓库档案 |

| 联系电话 | 文本 | 默认展示 |

## 四、新增 / 编辑页规范

### 4.1 字段控件与显隐规则

| 字段 | 控件 | 必填 | 显隐条件 |
| :--- | :--- | :--- | :--- |
| 仓库编码 | Input | 是 | 新增显示；编辑已引用时只读 |
| 仓库名称 | Input | 是 | 始终显示 |
| 仓库类型 | Select | 是 | 始终显示；编辑模式下只读（不允许修改） |
| 仓库地址 | TextArea | 是 | 始终显示 |
| 负责人 | Input | 是 | 始终显示 |
| 联系电话 | Input | 否 | 始终显示 |
| 仓库备注 | TextArea | 否 | 始终显示 |
| 停用原因 | TextArea | 条件必填 | 仅在停用弹窗中展示，不在主表单中出现 |

> **说明**：启停用状态不在新增/编辑主表单中以字段形式出现。启/停用操作通过列表页行操作按钮或详情页操作按钮触发确认弹窗完成，弹窗内含停用原因输入框（停用时）。

### 4.2 模式差异说明

| 比较维度 | 新增模式 | 编辑模式 | 查看模式 |
| :--- | :--- | :--- | :--- |
| 页面标题 | 新增仓库 | 编辑仓库 | 仓库详情 |
| 仓库编码 | 可编辑 | 已引用时只读 | 纯文本 |
| 仓库类型 | 可编辑 | 只读（一期不允许修改） | 纯文本 |
| 启停用状态 | 默认启用，新增时不展示状态字段 | 不在主表单展示，通过页面操作按钮触发 | 展示当前状态（StatusPill） |
| 系统信息 | 不展示 | 展示（只读） | 展示（只读） |

## 五、输入长度与格式限制

| 字段 | 长度限制 | 字符限制 | 补充说明 |
| :--- | :--- | :--- | :--- |
| 仓库编码 | 最多 20 个字符 | 仅允许字母、数字、连字符 `-` | 建议格式 `WH-0001` |
| 仓库名称 | 最多 50 个字符 | 禁止纯空白，默认不限制中文和常规符号 | 保存前自动去首尾空格 |
| 仓库类型 | 固定枚举 | 仅允许枚举值 | 后仓 / 门店仓 |
| 仓库地址 | 最多 100 个字符 | 允许中文、字母、数字和常规地址符号 | 默认单行高度，输入时自动增高 |
| 负责人 | 最多 20 个字符 | 禁止纯空白，默认允许中文、字母 | 保存前自动去首尾空格 |
| 联系电话 | 最多 20 个字符 | 手机号或座机格式 | 选填 |
| 仓库备注 | 最多 200 个字符 | 普通备注文本，不做特殊字符硬限制 | 默认单行高度，输入时自动增高 |
| 停用原因 | 最多 100 个字符 | 普通备注文本，不做特殊字符硬限制 | 状态=停用时必填 |

## 六、校验与保存规则

### 6.1 校验规则

| 校验点 | 规则 | 时机 |
| :--- | :--- | :--- |
| 仓库编码 | 必填，且必须唯一 | 保存时全量校验 |
| 仓库编码格式 | 仅允许字母、数字、连字符 `-` | 输入时 + 保存时 |
| 仓库名称 | 必填，最长 50 个字符 | 保存时全量校验 |
| 仓库类型 | 必填 | 保存时全量校验 |
| 仓库地址 | 必填，最长 100 个字符 | 保存时全量校验 |
| 负责人 | 必填，最长 20 个字符 | 保存时全量校验 |
| 联系电话 | 选填，填写时需符合手机号/座机格式 | 失焦或保存时校验 |
| 仓库备注 | 最长 200 个字符 | 输入时 + 保存时 |
| 停用原因 | 当状态=停用时必填，最长 100 个字符 | 保存时全量校验 |

补充说明：

- 编辑模式下，仓库编码字段必须以只读输入框或纯文本方式展示，不允许修改
- 前端不应把“仓库编码可编辑但保存时报错”作为交互方案

### 6.2 保存规则

| 场景 | 规则 |
| :--- | :--- |
| 新增保存成功 | 返回列表页，并在列表中看到最新新增数据 |
| 编辑保存成功 | 返回列表页，并在列表中看到最新编辑结果 |
| 保存失败 | 阻断提交，并定位首个报错字段 |
| 编辑仓库编码 | 不允许修改，不触发保存校验，直接按只读展示 |

## 七、反馈规范

| 操作 | 反馈方式 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 启用成功 | Toast | 仓库已启用 |
| 停用成功 | Toast | 仓库已停用 |
| 编码重复 | 字段红字提示 | 仓库编码已存在 |
| 引用后改编码 | Toast | 该仓库已被业务单据引用，编码不可修改 |
| 停用时未填写原因 | 弹窗内字段红字提示 | 停用原因不能为空 |

## 八、验收重点

- [ ] 列表默认列覆盖仓库识别与负责人信息
- [ ] 后仓 / 门店仓类型区分清晰

- [ ] 新增保存成功后返回列表页，并可看到最新新增数据
- [ ] 编辑保存成功后返回列表页，并可看到最新编辑结果
