# 仓库档案字段清单

> **文档定位**：仓库档案所有字段的唯一定义来源。下游文档只引用字段名称，不重复定义枚举值、必填性、计算规则。
>
> **引用规则**：
> - 销售出库、采购入库、调拨、盘点、报损、库存查询等页面引用仓库字段时，优先引用本清单口径
> - 下游单据中需要固化的仓库快照字段，标注「继承自仓库档案」，不重复写字段定义
> - 下游单据新增的执行态字段，只在对应单据自己的字段清单中定义
>
> **修改原则**：字段定义变化 → **只改本文件**；Demo PRD 不手动修改，由 AI 重新生成。
>
> **按需维度说明**：仓库档案一期无导入/导出功能规划，「可导入」「可导出」两列不加入本清单。
>
> **版本**：V1.1 | 更新时间：2026-04-19

## 五、展示与引用规则

- 仓库在所有下拉、搜索选择器中统一展示为 `仓库编码 仓库名称`。
- 仓库编码、仓库名称、仓库类型、负责人、启停用状态是列表页优先展示列。
- 销售订单、销售出库、采购入库、调拨单、盘点单、报损单需要引用仓库时，只带出仓库编码、仓库名称、仓库类型等基础识别信息。
- 停用仓库不允许在新建销售、采购、库存单据中继续选择；历史单据和库存查询仍允许查看。
- 为贴合课程一期口径，仓库档案先聚焦"货进到哪里、货从哪里出、谁负责这个仓"，不在主数据里展开能力开关和库存摘要。
