# 仓库档案主PRD

> **版本**：V1.0 | 2026-04-18
> **读者**：产品经理、前端工程师、测试工程师
> **说明**：仓库档案属于主数据 CRUD，当前补充模块级主PRD，用于统一仓库档案的业务边界、共性规则和验收重点。

---

## 1. 业务背景

强盛科技的一期进销存，不只是要记录“有几个仓”，而是要明确：

- 货从哪里出
- 货进到哪里
- 谁负责这个仓
- 前店后仓的仓库角色怎么区分

因此，仓库档案的目标是统一“仓的识别信息和业务角色”。

---

## 2. 功能范围

**In Scope**

- 仓库基础资料维护
- 仓库类型维护
- 负责人和联系方式维护
- 启用 / 停用管理

**Out of Scope**

- 复杂仓网规划
- 仓库能力开关
- 在途、冻结、虚拟仓等高级库存口径
- 库位管理

---

## 3. 模块定位

| 项目 | 内容 |
| :--- | :--- |
| 模块类型 | 主数据 CRUD |
| 核心职责 | 统一仓库识别口径和仓库角色口径 |
| 直接服务 | 销售出库、采购入库、调拨、盘点、报损 |
| 状态管理 | 启用 / 停用 |

---

## 4. 典型业务场景

| 场景ID | 场景 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| S01 | 新增后仓 | 主流程 | 新租用或新启用一个后仓，需要支持采购入库、销售出库、库存查询 |
| S02 | 新增门店仓 | 主流程 | 新开门店时建立门店仓，用于零售收银、调拨入库、门店库存展示 |
| S03 | 维护仓库信息 | 主流程 | 仓库地址、负责人、联系电话变化后，需要更新档案但不影响历史单据 |
| S04 | 停用仓库 | 辅助 | 门店关闭、仓库停用后，不再允许新单继续引用 |
| S05 | 恢复启用 | 辅助 | 已停用仓库恢复使用，重新允许在新单中选择 |
| S06 | 查看历史仓库 | 辅助 | 已停用仓库仍需在历史单据和查询中可见，保证链路追溯 |

## 5. 模块级通用规则

| 规则ID | 规则 |
| :--- | :--- |
| R01 | 仓库编码必须唯一，且一旦被单据引用后不可修改。 |
| R02 | 仓库必须至少维护编码、名称、类型、地址、负责人。 |
| R03 | 仓库类型当前只区分后仓和门店仓。 |
| R04 | 下游单据只引用仓库编码、名称、类型等基础识别信息。 |
| R05 | 停用仓库不可在新建销售、采购、库存单据中继续选择。 |

## 6. 状态与动作规则

| 状态 | 含义 |
| :--- | :--- |
| 启用 | 可在新业务中继续选择 |
| 停用 | 不可在新业务中继续选择 |

| 动作 | 启用 | 停用 | 说明 |
| :--- | :---: | :---: | :--- |
| 查看 | ✅ | ✅ | 始终允许 |
| 编辑 | ✅ | ✅ | 编码字段受引用约束 |
| 停用 | ✅ | ❌ | 停用时需填写原因 |
| 启用 | ❌ | ✅ | 支持恢复 |
| 删除 | ❌ | ❌ | 主数据不提供删除能力。若对象不再使用，统一通过“停用”处理；已被业务单据引用的数据必须长期保留，确保历史记录可追溯。 |

---

## 7. 动作级规则

### 7.1 新增规则

| 规则ID | 规则 |
| :--- | :--- |
| A01 | 新增仓库时必须填写：仓库编码、仓库名称、仓库类型、仓库地址、负责人。 |
| A02 | 仓库编码新增时必须做唯一性校验，重复则阻断保存。 |
| A03 | 联系电话和仓库备注可为空，空值统一按 `-` 展示。 |

### 7.2 编辑规则

| 规则ID | 规则 |
| :--- | :--- |
| A11 | 已被业务单据引用的仓库，编码不可编辑。 |
| A12 | 仓库名称、地址、负责人、联系电话、备注允许编辑。 |
| A13 | 仓库类型**一期不允许修改**。原因：仓库类型影响出入库单据的业务逻辑分区（后仓 vs 门店仓的盘点/出库规则不同），变更会导致历史数据口径混乱，需要评估影响范围后明确策略才可开放。如业务确有必要，应走"停用旧仓，新建正确类型仓库"的流程。 |

### 7.3 停用 / 启用规则

| 规则ID | 规则 |
| :--- | :--- |
| A21 | 停用仓库时必须填写停用原因。 |
| A22 | 停用后不可在新建销售、采购、库存单据中继续选择。 |
| A23 | 已停用仓库允许恢复启用，恢复后可重新在新单中选择。 |

## 8. 下游引用影响规则

| 引用模块 | 引用方式 | 说明 |
| :--- | :--- | :--- |
| 销售订单 | 选择默认发货仓 | 引用仓库编码、名称、类型 |
| 销售出库 | 出库执行仓 | 引用仓库编码、名称、类型 |
| 采购入库 | 入库目标仓 | 引用仓库编码、名称、类型 |
| 调拨单 | 调出仓 / 调入仓 | 引用仓库编码、名称、类型 |
| 盘点单 | 盘点仓 | 引用仓库编码、名称、类型 |
| 报损单 | 报损仓 | 引用仓库编码、名称、类型 |

补充规则：

- 已被业务单据引用的仓库，历史单据继续展示原仓库信息
- 停用仓库不影响历史单据查看和统计
- 当前一期下游单据优先引用基础识别信息，不展开复杂仓库能力字段

## 9. 异常与边界规则

| 场景 | 处理口径 |
| :--- | :--- |
| 仓库编码重复 | 阻断保存，提示“仓库编码已存在” |
| 仓库名称重复但编码不同 | 当前一期允许，但建议人工确认是否重复建仓 |
| 停用时未填写原因 | 阻断停用，提示补充停用原因 |
| 已停用仓库被历史单据查询命中 | 允许展示，并在状态上明确标识“停用” |
| 已引用仓库尝试修改编码 | 阻断修改，提示该仓库已被业务单据引用 |
| 仓库类型变更 | 当前一期默认允许，但应提示评估对业务理解的影响 |

## 11. 反馈与验收重点

### 11.1 关键反馈

| 操作 | 反馈方式 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 启用成功 | Toast | 启用成功 |
| 停用成功 | Toast | 停用成功 |
| 编码重复 | 字段内红字提示 | 仓库编码已存在 |
| 已引用不可改编码 | Toast | 该仓库已被业务单据引用，编码不可修改 |

### 11.2 验收重点

| # | 验收项 | 验收标准 |
| :--- | :--- | :--- |
| V01 | 编码唯一性 | 仓库编码不允许重复 |
| V02 | 仓库角色清晰 | 后仓 / 门店仓区分明确 |
| V03 | 引用后锁定 | 已引用仓库不可修改编码 |
| V04 | 停用限制 | 停用仓库不可继续在新单中选择 |
| V05 | 页面分组合理 | 基础资料、说明、系统信息分组清晰 |
