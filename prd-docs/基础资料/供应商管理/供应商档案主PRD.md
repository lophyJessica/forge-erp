# 供应商档案主PRD

> **版本**：V1.1 | 2026-04-19
> **读者**：产品经理、前端工程师、测试工程师
> **说明**：供应商档案属于主数据 CRUD，当前补充模块级主PRD，用于统一供应商档案的业务边界、共性规则和验收重点。

---

## 1. 业务背景

采购链路能不能稳定跑通，取决于供应商是否被统一管理。  
如果供应商资料分散在聊天记录或 Excel 里，就会出现：

- 向谁买不清楚
- 联系人和电话不统一
- 结算方式、账期、税率口径混乱

因此，供应商档案的目标是统一"向谁买、怎么联系、怎么结算"。

---

## 2. 功能范围

**In Scope**

- 供应商基础资料维护
- 结算方式、账期、税率维护
- 启用 / 停用管理
- 详情查看

**Out of Scope**

- 供应商评级体系
- 供应商协同门户
- 对账单自动回传
- 复杂税务模型

## 4. 典型业务场景

| 场景ID | 场景 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| S01 | 新增供应商档案 | 主流程 | 新增合作供应商时建立编码、联系人、结算规则等基础信息 |
| S02 | 维护结算规则 | 主流程 | 更新供应商结算方式、账期、税率等采购规则 |
| S03 | 更新联系方式 | 主流程 | 联系人、联系电话、联系地址变化后更新档案 |
| S04 | 停用供应商 | 辅助 | 供应商不再合作时停用，禁止新单继续引用 |
| S05 | 恢复启用 | 辅助 | 已停用供应商恢复合作后重新启用 |
| S06 | 查看历史供应商 | 辅助 | 已停用供应商仍需在历史采购单据和应付查询中可见 |

## 5. 核心业务规则

| 规则ID | 规则 |
| :--- | :--- |
| R01 | 供应商编码必须唯一，且一旦被采购单据引用后不可修改。 |
| R02 | 供应商必须维护联系人、联系电话、结算方式。账期天数仅在"账期结算"时必填；税率为选填字段，可后续补充维护。 |
| R03 | 账期结算供应商必须填写账期天数；现付供应商默认账期天数为 `0`，且不可手动编辑。 |
| R04 | 采购订单引用供应商时，需要固化供应商名称、联系人、联系电话、结算方式和账期快照。 |
| R05 | 停用供应商不可在新建采购订单、采购入库、付款登记中继续选择。 |

## 6. 状态与动作规则

| 状态 | 含义 |
| :--- | :--- |
| 启用 | 可在新业务中继续选择 |
| 停用 | 不可在新业务中继续选择 |

| 动作 | 启用 | 停用 | 说明 |
| :--- | :---: | :---: | :--- |
| 查看 | ✅ | ✅ | 始终允许 |
| 编辑 | ✅ | ✅ | 编码字段受引用约束 |
| 停用 | ✅ | ❌ | 停用时需填写原因 |
| 启用 | ❌ | ✅ | 支持恢复 |
| 删除 | ❌ | ❌ | 主数据不提供删除能力。若对象不再使用，统一通过"停用"处理；已被业务单据引用的数据必须长期保留，确保历史记录可追溯。 |

---

## 7. 动作级规则

### 7.1 新增规则

| 规则ID | 规则 |
| :--- | :--- |
| A01 | 新增供应商时必须填写：供应商编码、供应商名称、联系人、联系电话、结算方式。 |
| A02 | 供应商编码新增时必须做唯一性校验，重复则阻断保存。 |
| A03 | 联系地址、税率为可选字段，允许后续补充维护。 |
| A04 | 新增时供应商默认为"启用"状态，不允许在新增表单中直接选择状态。 |

### 7.2 编辑规则

| 规则ID | 规则 |
| :--- | :--- |
| A11 | 已被采购单据引用的供应商，编码不可编辑。 |
| A12 | 供应商名称、联系人、联系电话、联系地址、结算方式、账期、税率允许编辑。 |
| A13 | 已停用供应商仍允许查看和编辑非编码类字段。 |

### 7.3 停用 / 启用规则

| 规则ID | 规则 |
| :--- | :--- |
| A21 | 停用供应商时必须填写停用原因。 |
| A22 | 停用后不可在新建采购订单、采购入库、付款登记中继续选择。 |
| A23 | 已停用供应商允许恢复启用，恢复后可重新在新单中选择。 |
| A24 | **启停用操作通过独立的"停用/启用"动作按钮触发，不能在编辑表单中直接修改状态字段。** 停用时系统弹出确认弹窗，要求用户填写停用原因后方可完成操作。 |

## 8. 下游引用影响规则

| 引用模块 | 引用方式 | 说明 |
| :--- | :--- | :--- |
| 采购订单 | 选择供应商 | 引用供应商编码、名称、联系人、联系电话、结算方式、账期快照 |
| 采购入库 | 来源采购单或手工选择供应商 | 引用供应商编码、名称、联系人、结算方式 |
| 采购退货 | 关联原采购入库单 | 引用供应商编码、名称、联系人 |
| 付款登记 | 选择供应商 | 引用供应商编码、名称、结算方式 |

补充规则：

- 已被业务单据引用的供应商，历史单据继续展示原供应商信息
- 停用供应商不影响历史采购单据和应付查询展示
- 当前一期不在供应商档案中展开供应商评级、对账协同和门户能力

## 9. 异常与边界规则

| 场景 | 处理口径 |
| :--- | :--- |
| 供应商编码重复 | 阻断保存，提示"供应商编码已存在" |
| 停用时未填写原因 | 阻断停用，弹窗内提示补充停用原因 |
| 已引用供应商尝试修改编码 | 阻断修改，提示该供应商已被业务单据引用 |
| 结算方式改为现付 | 自动将账期天数重置为 `0`，并置为只读展示 |
| 供应商名称重复但编码不同 | 当前一期允许，但建议人工确认是否重复建档 |
| 税率为空 | 允许保存；后续采购对账时再补充维护 |

## 10. 权限设计

> **课程口径说明**：一期进销存课程场景较小，采购和财务均有较大自主权，故允许两个角色进行供应商新增和编辑。实际企业场景中，供应商档案通常需要走审批流或限制为采购主管/管理员才能维护。本模块的权限设计为课程简化版，不作为真实系统设计参考。

| 操作 | 采购 | 财务 | 管理员 |
| :--- | :---: | :---: | :---: |
| 查看 | ✅ | ✅ | ✅ |
| 新增 | ✅ | ✅ | ✅ |
| 编辑 | ✅ | ✅ | ✅ |
| 启用/停用 | ❌ | ❌ | ✅ |

## 11. 反馈与验收重点

### 11.1 关键反馈

| 操作 | 反馈方式 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 启用成功 | Toast | 供应商已启用 |
| 停用成功 | Toast | 供应商已停用 |
| 编码重复 | 字段内红字提示 | 供应商编码已存在 |
| 已引用不可改编码 | Toast | 该供应商已被业务单据引用，编码不可修改 |
| 结算方式切换 | 联动更新 | 切换结算方式后同步刷新账期字段 |
| 停用时原因未填 | 弹窗内字段红字提示 | 停用原因不能为空 |

### 11.2 验收重点

| # | 验收项 | 验收标准 |
| :--- | :--- | :--- |
| V01 | 编码唯一性 | 供应商编码不允许重复 |
| V02 | 结算规则完整 | 结算方式、账期、税率字段齐全且必填逻辑正确 |
| V03 | 条件联动 | 账期结算与账期天数字段联动正确；现付时账期自动置 `0` 且只读 |
| V04 | 快照规则明确 | 采购订单引用供应商后快照口径清晰 |
| V05 | 停用限制 | 停用供应商不可继续用于新单 |
| V06 | 启停用操作路径 | 启停用通过操作按钮弹窗触发，不在编辑表单中出现状态字段 |
