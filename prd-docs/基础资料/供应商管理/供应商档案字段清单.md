# 供应商档案字段清单

> **文档定位**：供应商档案所有字段的唯一定义来源。下游文档只引用字段名称，不重复定义枚举值、必填性、计算规则。
>
> **引用规则**：
> - 采购订单、采购入库、采购退货、付款登记等页面引用供应商字段时，优先引用本清单口径
> - 下游单据中需要固化的供应商快照字段，标注「继承自供应商档案」，不重复写字段定义
> - 下游单据新增的执行态字段，只在对应单据自己的字段清单中定义
>
> **修改原则**：字段定义变化 → **只改本文件**；Demo PRD 不手动修改，由 AI 重新生成。
>
> **按需维度说明**：供应商档案一期无导入/导出功能规划，「可导入」「可导出」两列不加入本清单。
>
> **版本**：V1.1 | 更新时间：2026-04-19

## 五、展示与引用规则

- 供应商在所有下拉、搜索选择器中统一展示为 `供应商编码 供应商名称`。
- 供应商编码、供应商名称、联系人、联系电话、结算方式、账期天数、税率、启停用状态是列表页优先展示列。
- 采购订单需要固化的供应商字段包括供应商名称、联系人、联系电话、结算方式、账期天数；下游单据引用时按快照处理。
- 停用供应商不允许在新建采购订单、采购入库、付款登记中继续选择；历史采购单据和应付查询仍允许查看。
- 结算方式为 `ACCOUNT` 时，账期天数必须填写；为 `CASH` 时默认展示 `0`。
- 为贴合课程一期口径，供应商档案先聚焦"向谁买、怎么联系、怎么结算、账期多久、税率多少"，不在主数据里展开往来摘要和复杂协同字段。
