# 客户档案字段清单

> **文档定位**：客户档案所有字段的唯一定义来源。下游文档只引用字段名称，不重复定义枚举值、必填性、计算规则。
>
> **引用规则**：
> - 销售订单、收款登记、客户往来查询等页面引用客户字段时，优先引用本清单口径
> - 下游单据中需要固化的客户快照字段，标注「继承自客户档案」，不重复写字段定义
> - 下游单据新增的执行态字段，只在对应单据自己的字段清单中定义
>
> **修改原则**：字段定义变化 → **只改本文件**；Demo PRD 不手动修改，由 AI 重新生成。
>
> **按需维度说明**：客户档案一期无导入/导出功能规划，「可导入」「可导出」两列不加入本清单。
>
> **版本**：V1.1 | 更新时间：2026-04-19

## 五、展示与引用规则

- 客户在所有下拉、搜索选择器中统一展示为 `客户编码 客户名称`。
- 客户编码、客户名称、客户等级、联系人、联系电话、价格级别、结算方式、账期天数、信用额度、启停用状态是列表页优先展示列。
- 销售订单需要固化的客户字段包括客户名称、联系人、联系电话、收货信息、价格级别、结算方式、账期天数；下游单据引用时按快照处理。
- 停用客户不允许在新建销售订单和收款登记中继续选择；历史订单、往来查询和统计页面仍允许查看。
- 结算方式为 `CASH` 时，账期天数固定展示 `0`，信用额度默认 `0.00`；允许编辑为账期结算后再填写完整额度信息。
- 为贴合课程一期口径，客户档案先聚焦"客户是谁、什么价格、什么账期、多少信用额度"，不在主数据里展开往来摘要和复杂归属字段。
