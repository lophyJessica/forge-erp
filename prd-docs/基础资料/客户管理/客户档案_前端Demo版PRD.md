# 客户档案_前端Demo版PRD

**version**：v1.2.0  
**date**：2026-04-19  
**description**：客户档案前端页面规范，统一承接客户列表页、查看页、新增页、编辑页。字段定义以《客户档案字段清单》为准；本文档只定义页面结构、控件类型、显隐逻辑、交互与 Mock 数据要求。

---

## 一、页面清单

| 页面 | 目标 |
| :--- | :--- |
| 客户列表页 | 查询客户、查看状态、进入新增/编辑/详情 |
| 客户详情页 | 查看客户基础资料、价格与往来规则 |
| 新增客户页 | 创建新客户档案 |
| 编辑客户页 | 修改已有客户档案 |

---

## 二、列表页规范

### 2.1 查询条件区

| 字段 | 控件类型 | 说明 |
| :--- | :--- | :--- |
| 综合搜索 | Input | 支持客户编码 / 客户名称 / 联系人 |
| 客户等级 | Select | 全部 / A级 / B级 / C级 |
| 结算方式 | Select | 全部 / 现结 / 账期结算 |
| 状态 | Select | 全部 / 启用 / 停用 |

### 2.2 表格列定义

| 列名 | 展示规则 | 说明 |
| :--- | :--- | :--- |
| 客户编码 | 文本 | 默认展示 |
| 客户名称 | 文本 | 默认展示 |
| 客户等级 | 文本 | 默认展示 |
| 联系人 | 文本 | 默认展示 |
| 联系电话 | 文本 | 默认展示 |
| 价格级别 | 文本 | 默认展示 |
| 结算方式 | 文本 | 默认展示 |
| 账期天数 | 数值 | 默认展示 |
| 信用额度 | 金额 | 保留2位小数 |
| 状态 | StatusPill | 启用=绿色 / 停用=灰色 |
| 操作 | 固定右侧 | 查看 / 编辑 / 停用（启用态）/ 启用（停用态） |

> **操作列说明**：启停用通过行操作按钮触发确认弹窗，不在主表单中操作。停用弹窗内含"停用原因"必填输入框；启用弹窗为纯确认。

## 三、查看页规范

### 3.1 信息分组

| 分组 | 字段 |
| :--- | :--- |
| 客户资料 | 客户编码、客户名称、客户等级、联系人、联系电话 |
| 往来规则 | 价格级别、结算方式、账期天数、信用额度 |
| 收货信息 | 默认收货人、默认收货电话、默认收货地址 |
| 补充信息 | 客户备注 |
| 系统信息 | 创建人、创建时间、最后修改人、最后修改时间 |

> **空值展示**：所有字段无数据时统一展示 `-`，不留空白。  
> **停用原因**：仅当客户状态为"停用"时，在"补充信息"分组内展示停用原因；启用状态下展示 `-`。

## 四、新增 / 编辑页规范

### 4.1 字段控件与显隐规则

| 字段 | 控件 | 必填 | 显隐条件 |
| :--- | :--- | :--- | :--- |
| 客户编码 | Input | 是 | 新增显示；编辑已引用时只读 |
| 客户名称 | Input | 是 | 始终显示 |
| 客户等级 | Select | 是 | 始终显示 |
| 联系人 | Input | 是 | 始终显示 |
| 联系电话 | Input | 是 | 始终显示 |
| 价格级别 | Select | 是 | 始终显示 |
| 结算方式 | Select | 是 | 始终显示 |
| 账期天数 | Input（正整数） | 条件必填 | 结算方式=账期结算时显示为必填；现结时只读展示 `0` |
| 信用额度 | Input（≥0，保留2位小数） | 条件必填 | 结算方式=账期结算时显示为必填；现结时只读展示 `0.00` |
| 默认收货人 | Input | 否 | 始终显示 |
| 默认收货电话 | Input | 否 | 始终显示 |
| 默认收货地址 | TextArea | 否 | 始终显示 |
| 客户备注 | TextArea | 否 | 始终显示 |
| 停用原因 | TextArea | 条件必填 | **仅在停用弹窗中展示**，不在主表单中出现 |

> **说明**：启停用状态不在新增/编辑主表单中以字段形式出现。启/停用操作通过列表页行操作按钮或详情页操作按钮触发确认弹窗完成，弹窗内含停用原因输入框（停用时必填）。

### 4.2 模式差异说明

| 比较维度 | 新增模式 | 编辑模式 | 查看模式 |
| :--- | :--- | :--- | :--- |
| 页面标题 | 新增客户 | 编辑客户 | 客户详情 |
| 客户编码 | 可编辑 | 已引用时只读 | 纯文本 |
| 账期天数/信用额度 | 跟随结算方式联动 | 跟随结算方式联动 | 纯文本 |
| 启停用状态 | 默认启用，新增时不展示状态字段 | 不在主表单展示，通过页面操作按钮触发 | 展示当前状态（StatusPill） |
| 系统信息 | 不展示 | 展示（只读） | 展示（只读） |

## 五、输入长度与格式限制

| 字段 | 长度限制 | 字符限制 | 补充说明 |
| :--- | :--- | :--- | :--- |
| 客户编码 | 最多 20 个字符 | **仅允许**大写字母（A-Z）、数字（0-9）、连字符（-）；**禁止**中文、空格及其他特殊字符 | 参考格式 `CUST-0001`；最短 2 个字符 |
| 客户名称 | 最多 50 个字符 | 禁止纯空白，允许中文和常规符号 | 保存前自动去首尾空格 |
| 客户等级 | 固定枚举 | A级 / B级 / C级 | 必填 |
| 联系人 | 最多 20 个字符 | 禁止纯空白，允许中文、字母 | 保存前自动去首尾空格 |
| 联系电话 | 最多 20 个字符 | 手机号或座机格式 | 手机11位/座机含区号 |
| 价格级别 | 固定枚举 | 一级价 / 二级价 / 三级价 | 必填 |
| 结算方式 | 固定枚举 | 现结 / 账期结算 | 必填 |
| 账期天数 | - | 整数，0-365 | 账期结算时必填 |
| 信用额度 | - | ≥ 0，保留 2 位小数 | 账期结算时必填 |
| 默认收货人 | 最多 20 个字符 | 禁止纯空白，允许中文、字母 | 选填 |
| 默认收货电话 | 最多 20 个字符 | 手机号或座机格式 | 选填 |
| 默认收货地址 | 最多 100 个字符 | 允许中文、字母、数字和常规地址符号 | 选填 |
| 客户备注 | 最多 200 个字符 | 普通备注文本，不做特殊字符硬限制 | 选填 |
| 停用原因 | 最多 100 个字符 | 普通备注文本，不做特殊字符硬限制 | 停用时弹窗内必填 |

## 六、校验与保存规则

### 6.1 校验规则

| 校验点 | 规则 | 时机 |
| :--- | :--- | :--- |
| 客户编码（必填/唯一） | 必填；最多 20 个字符；全局唯一 | 保存时全量校验 |
| 客户编码（格式） | 仅允许大写字母（A-Z）、数字（0-9）、连字符（-）；不允许中文、空格及其他字符；输入非法字符时**立即阻断**，不允许录入 | 输入时实时阻断 + 保存时全量校验 |
| 客户名称 | 必填，最长 50 个字符 | 保存时全量校验 |
| 客户等级 | 必填 | 保存时全量校验 |
| 联系人 | 必填，最长 20 个字符 | 保存时全量校验 |
| 联系电话 | 必填，手机号或座机格式 | 失焦或保存时校验 |
| 价格级别 | 必填 | 保存时全量校验 |
| 结算方式 | 必填 | 保存时全量校验 |
| 账期天数 | 结算方式=账期结算时必填；取值范围 0-365 整数 | 保存时全量校验 |
| 信用额度 | 结算方式=账期结算时必填；≥ 0；保留 2 位小数 | 保存时全量校验 |
| 停用原因（弹窗） | 停用动作触发时必填，最长 100 个字符 | 弹窗确认时校验 |

补充说明：

- 编辑模式下，客户编码字段以只读输入框或纯文本方式展示，不允许修改（已被业务单据引用时）
- 结算方式=账期结算时，账期天数和信用额度切换为可编辑；切换为现结时，两个字段自动重置为 `0` / `0.00` 且只读展示，不参与保存校验

### 6.2 保存规则

| 场景 | 规则 |
| :--- | :--- |
| 新增保存成功 | 返回列表页，并在列表中看到最新新增数据 |
| 编辑保存成功 | 返回列表页，并在列表中看到最新编辑结果 |
| 保存失败 | 阻断提交，定位首个报错字段 |
| 编码已引用 | 不允许修改编码，不触发唯一性校验，直接以只读展示 |

## 七、反馈规范

| 操作 | 反馈方式 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 启用成功 | Toast | 客户已启用 |
| 停用成功 | Toast | 客户已停用 |
| 编码格式非法 | 字段红字提示（实时） | 客户编码只允许大写字母、数字和连字符（-） |
| 编码重复 | 字段红字提示 | 客户编码已存在 |
| 引用后改编码 | Toast | 该客户已被业务单据引用，编码不可修改 |
| 结算方式切换 | 联动更新 | 账期天数和信用额度切换为只读/可编辑；切换为现结时自动重置为 `0` 和 `0.00` |
| 停用时未填写原因 | 弹窗内字段红字提示 | 停用原因不能为空 |

## 八、验收重点

- [ ] 列表默认列覆盖客户识别、价格与信用口径
- [ ] 结算方式与账期、信用额度联动正确（切换现结时自动清零且只读展示）
- [ ] 已引用客户的编码不可编辑
- [ ] 停用客户不能继续在新单中选择
- [ ] 启停用通过操作按钮弹窗触发，弹窗内停用原因必填校验正确
- [ ] 客户备注、收货信息等选填字段空值展示 `-`
- [ ] 新增保存成功后返回列表页，并可看到最新新增数据
- [ ] 编辑保存成功后返回列表页，并可看到最新编辑结果
