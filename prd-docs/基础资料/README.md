## 文档工作流

基础资料按“一个模块一个文档组”的方式维护：

```text
字段清单（SST） → 主PRD（模块边界/共性规则） → 前端Demo PRD → 前端原型
```

主数据属于简单 CRUD，标准做法仍是 **字段清单 + 前端Demo PRD**。  
当前额外补充“模块级主PRD”，目的是把主数据的业务边界、启停用规则、权限和验收口径写清楚，避免这些内容分散在字段清单里。

## 当前补充口径

- 表格结构统一参考 `通用资产库/02-文档模板/字段清单模板.md`

- 各模块文档独立维护，后续若新增基础资料模块，直接新增子文件夹即可
