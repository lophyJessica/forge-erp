# 商品档案字段清单

> **文档定位**：商品档案所有字段的唯一定义来源。下游文档只引用字段名称，不重复定义枚举值、必填性、计算规则。
>
> **引用规则**：
> - 销售订单、采购订单、零售收银、库存查询等下游页面引用商品字段时，优先引用本清单口径
> - 下游单据中需要固化的商品快照字段，标注「继承自商品档案」，不重复写字段定义
> - 下游单据新增的执行态字段，只在对应单据自己的字段清单中定义
>
> **修改原则**：字段定义变化 → **只改本文件**；Demo PRD 不手动修改，由 AI 重新生成。
>
> **按需维度说明**：商品档案一期无导入/导出功能规划，「可导入」「可导出」两列不加入本清单。
>
> **版本**：V1.3 | 更新时间：2026-07-13

## 四、展示与引用规则

- 商品在所有下拉、搜索选择器中统一展示为 `商品编码 商品名称`。
- 商品编码、商品名称、商品分类、规格型号、销售单位、安全库存、默认批发价、启停用状态是列表页优先展示列；安全库存未维护时展示 `-`。
- 商品被销售订单、采购订单、零售收银、库存流水引用后，不允许修改商品编码；其他字段允许编辑，但下游单据仅继承生成当时固化的快照字段。
- 启停用状态为 `DISABLED` 的商品，不允许在新建销售订单、零售收银、采购订单中继续选择；历史单据和库存查询仍可查看。
- 条码字段只用于零售和快捷搜索，不作为商品主键；禁止一个条码对应多个启用商品。
- 一期可维护「安全库存」选填数值，供即时库存查询只读 Tag 对比；不承载库存摘要、自动预警推送、多单位换算（后两者见库存/二期模块）。
