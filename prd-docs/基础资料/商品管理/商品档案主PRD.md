# 商品档案主PRD

> **版本**：V1.2 | 2026-05-26
> **读者**：产品经理、前端工程师、测试工程师
> **说明**：商品档案属于主数据 CRUD，当前补充模块级主PRD，用于统一商品档案的业务边界、共性规则和验收重点。

---

## 1. 业务背景

强盛科技在渠道扩张期已经同时承接批发、零售和采购补货。  
商品如果没有统一的编码、名称、规格和价格口径，就会直接导致：

- 同一商品不同人叫法不一致
- 批发报价、零售售价、采购参考价无法统一
- 销售、采购、库存三条链路很难围绕同一商品对象协同

因此，商品档案的核心目标是先统一"卖什么、买什么、按什么规格和价格承接"。

---

## 2. 功能范围

**In Scope**

- 商品基础信息维护
- 商品价格信息维护（批发价、零售价、参考采购价）
- **安全库存**（选填数值，供「即时库存查询」只读对比展示，不做自动预警推送）
- 启用 / 停用管理
- 商品详情查看
- 下游单据引用时的快照口径约束

**Out of Scope**

- 品牌主数据管理
- 多级分类树
- 多单位换算
- 安全库存自动预警推送、补货建议、消息通知（属于二期；一期仅维护数值）
- 批次、序列号、效期体系

## 4. 典型业务场景

| 场景ID | 场景 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| S01 | 新增商品档案 | 主流程 | 新上架商品时维护编码、名称、分类、规格和价格信息 |
| S02 | 编辑商品信息 | 主流程 | 商品名称、规格、价格、备注等可变信息更新 |
| S03 | 停用商品 | 辅助 | 商品停售或不再经营时停用，禁止新单继续引用 |
| S04 | 恢复启用 | 辅助 | 停用商品重新恢复销售或采购时恢复启用 |
| S05 | 查看历史商品 | 辅助 | 已停用商品仍需在历史单据和库存查询中可见 |
| S06 | 零售与批发共用商品 | 主流程 | 同一商品同时服务批发与零售，需要统一编码和价格口径 |

## 5. 核心业务规则

| 规则ID | 规则 |
| :--- | :--- |
| R01 | 商品编码必须唯一，且一旦被下游单据引用后不可修改。 |
| R02 | 商品至少维护编码、名称、分类、规格型号、销售单位和默认批发价。 |
| R03 | 默认批发价为必填；默认零售价仅当商品参与零售时必填，其他场景选填；参考采购价始终为选填。 |
| R04 | 条码字段主要服务零售收银与快捷搜索，不作为主键；禁止一个条码对应多个启用商品。 |
| R05 | 停用商品不可在新建销售订单、采购订单、零售收银中继续选择，但历史单据仍允许展示。 |

## 6. 状态与动作规则

| 状态 | 含义 |
| :--- | :--- |
| 启用 | 可在新单据中继续选择 |
| 停用 | 不可在新单据中继续选择 |

| 动作 | 启用 | 停用 | 说明 |
| :--- | :---: | :---: | :--- |
| 查看 | ✅ | ✅ | 始终允许 |
| 编辑 | ✅ | ✅ | 编码字段受引用约束 |
| 停用 | ✅ | ❌ | 停用时需填写原因 |
| 启用 | ❌ | ✅ | 支持恢复 |
| 删除 | ❌ | ❌ | 主数据不提供删除能力。若对象不再使用，统一通过"停用"处理；已被业务单据引用的数据必须长期保留，确保历史记录可追溯。 |

## 7. 动作级规则

### 7.1 新增规则

| 规则ID | 规则 |
| :--- | :--- |
| A01 | 新增商品时必须填写：商品编码、商品名称、商品分类、规格型号、销售单位、默认批发价。 |
| A02 | 商品编码新增时必须做唯一性校验，重复则阻断保存。 |
| A03 | 默认零售价、参考采购价、条码为可选字段，允许后续补充维护。 |
| A04 | 新增时商品默认为"启用"状态，不允许在新增表单中直接选择状态。 |

### 7.2 编辑规则

| 规则ID | 规则 |
| :--- | :--- |
| A11 | 已被下游单据引用的商品，编码不可编辑。 |
| A12 | 商品名称、分类、规格型号、销售单位、价格、备注允许编辑。 |
| A13 | 条码允许编辑，但修改后必须重新通过唯一性校验。 |

### 7.3 停用 / 启用规则

| 规则ID | 规则 |
| :--- | :--- |
| A21 | 停用商品时必须填写停用原因。 |
| A22 | 停用后不可在新建销售订单、采购订单、零售收银中继续选择。 |
| A23 | 已停用商品允许恢复启用，恢复后可重新在新单中选择。 |
| A24 | **启停用操作通过独立的"停用/启用"动作按钮触发，不能在编辑表单中直接修改状态字段。** 停用时系统弹出确认弹窗，要求用户填写停用原因后方可完成操作。 |

## 8. 下游引用影响规则

| 引用模块 | 引用方式 | 说明 |
| :--- | :--- | :--- |
| 销售订单 | 选择销售商品 | 引用商品编码、名称、规格、单位、默认批发价快照 |
| 零售收银 | 扫码 / 搜索商品 | 引用商品编码、名称、规格、单位、默认零售价快照 |
| 采购订单 | 选择采购商品 | 引用商品编码、名称、规格、单位、参考采购价 |
| 库存查询 | 作为库存主体 | 引用商品编码、名称、规格、单位 |

补充规则：

- 已被业务单据引用的商品，历史单据继续展示原商品信息
- 停用商品不影响历史单据和库存查询展示
- 当前一期不在商品档案中展开库存摘要、预警值、多单位换算

## 9. 异常与边界规则

| 场景 | 处理口径 |
| :--- | :--- |
| 商品编码重复 | 阻断保存，提示"商品编码已存在" |
| 条码重复 | 阻断保存，提示"商品条码已存在" |
| 停用时未填写原因 | 阻断停用，弹窗内提示补充停用原因 |
| 已引用商品尝试修改编码 | 阻断修改，提示该商品已被业务单据引用 |
| 商品名称重复但编码不同 | 当前一期允许，但建议人工确认是否重复建档 |
| 默认零售价为空 | 允许保存；如需参与零售，再补充维护 |
| 参考采购价为空 | 允许保存；采购下单时手工录入 |

## 10. 权限设计

> **课程口径说明**：一期进销存课程中，商品档案由业务/采购共同维护，仓管只需查看。实际企业场景中，商品档案的价格字段通常需要商品经理或运营主管才能修改。本模块的权限设计为课程简化版，不作为真实系统设计参考。

| 操作 | 业务员/内勤 | 采购 | 仓管 | 管理员 |
| :--- | :---: | :---: | :---: | :---: |
| 查看 | ✅ | ✅ | ✅ | ✅ |
| 新增 | ✅ | ✅ | ❌ | ✅ |
| 编辑 | ✅ | ✅ | ❌ | ✅ |
| 启用/停用 | ❌ | ❌ | ❌ | ✅ |

## 11. 反馈与验收重点

### 11.1 关键反馈

| 操作 | 反馈方式 | 文案 |
| :--- | :--- | :--- |
| 保存成功 | Toast | 保存成功 |
| 启用成功 | Toast | 商品已启用 |
| 停用成功 | Toast | 商品已停用 |
| 编码重复 | 字段内红字提示 | 商品编码已存在 |
| 条码重复 | 字段内红字提示 | 商品条码已存在 |
| 已引用不可改编码 | Toast | 该商品已被业务单据引用，编码不可修改 |
| 停用时原因未填 | 弹窗内字段红字提示 | 停用原因不能为空 |

### 11.2 验收重点

| # | 验收项 | 验收标准 |
| :--- | :--- | :--- |
| V01 | 编码唯一性 | 商品编码不允许重复 |
| V02 | 条码唯一性 | 启用商品的条码不允许重复 |
| V03 | 引用后锁定 | 已引用商品编码不可修改 |
| V04 | 价格口径完整 | 默认批发价、默认零售价、参考采购价必填/选填逻辑正确 |
| V05 | 停用限制 | 停用商品不可在新单中继续选择 |
| V06 | 启停用操作路径 | 启停用通过操作按钮弹窗触发，不在编辑表单中出现状态字段 |
